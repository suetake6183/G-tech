# インボックス（未分類データ置き場）

**用途**: どこに置けばいいかわからないデータを一時的に保管

---

## このフォルダの役割

プロジェクト関連、クライアント関連、個人思考など、**分類に迷った時の一時保管場所**です。

### 使うべきタイミング
- 「どこに置くべきか判断できない」
- 「とりあえず記録しておきたい」
- 「後で整理する前提で保存したい」

### 使わないべきタイミング
- プロジェクトが明確 → `02_クライアント情報/[クライアント]/[プロジェクト]/00_INPUT/`
- クライアント関連（プロジェクト外） → `02_クライアント情報/[クライアント]/00_CORE/source/`
- 個人思考 → `03_思考と探求/_inbox/`
- リサーチ結果 → `01_ナレッジベース/_inbox/`

---

## ファイル命名規則

```
YYYY-MM-DD_[内容].[拡張子]

例:
2025-11-18_新規ツールリサーチ.md
2025-11-20_アイデアメモ.md
```

---

## YAMLメタデータ（必須）

ファイルの冒頭に必ず記載：

```yaml
---
type: [email / meeting / document / voice_memo / research / other]
date: YYYY-MM-DD
topics: [トピック1, トピック2]
processing_status: raw  # raw / reviewed / processed / archived
---
```

**なぜ必須か**: AIがファイルを読んだ時に「何について、どう扱うべきか」を理解するため

---

## 処理方法

### `/process` コマンドで整理

```
/process
```

Claudeが以下をサポート：
1. ファイルの内容を理解
2. 適切な配置場所を提案
3. ナレッジ化・ブログ化の判断サポート

### 手動で移動

適切な場所がわかったら、手動で移動してもOK。

---

## 運用ルール

1. **溜めすぎない**: 月1回は整理
2. **メタデータは必須**: 後で理解できるように
3. **完璧を求めない**: 80%正確なら十分、後で修正可能

---

## テンプレート

`_TEMPLATE/` フォルダに以下のテンプレートがあります：

- `EMAIL_TEMPLATE.md` - メール記録用
- `VOICE_MEMO_TEMPLATE.md` - 音声文字起こし用
- `RESEARCH_TEMPLATE.md` - リサーチ結果用
- `MEETING_TEMPLATE.md` - 議事録用

コピーして使ってください。

---

**原則**: 迷ったらここに入れる。間違った場所より、インボックスの方がマシ。
