# クライアント視点の重要性

**作成日**: 2025年11月14日
**きっかけ**: 黒澤工務店への提案書作成時のフィードバックから

---

## 問題の発見

提案書で「リスクを負っています」「私も報酬を受け取りません」といった表現を繰り返していた。

Claude Codeからの質問：
> リスクを負っているという言い方を相手にする必要はあるでしょうか？
> これを聞いて相手はどう思うでしょうか？

この問いで気づいた：**焦点が間違っている**

---

## 提供者視点 vs クライアント視点

### 提供者視点（NG）

**特徴**:
- 「私がこんなに頑張っている」
- 「私もリスクを負っている」
- 「私も損をする」

**なぜダメか**:
1. **焦点が間違っている**
   - クライアントは提供者の苦労に関心がない
   - 自分にとってのメリットを知りたい

2. **自己憐憫に聞こえる**
   - 「こんなに頑張っているのだから、成果が出たら払ってね」
   - ギルトトリップ（罪悪感の押し付け）的

3. **プロフェッショナルではない**
   - プロは自分の苦労を見せない
   - クライアントに価値を提供することに集中する

---

### クライアント視点（OK）

**特徴**:
- 「御社のリスクはゼロです」
- 「御社の費用負担は一切ありません」
- 「御社は初期投資なしで始められます」

**なぜ良いか**:
1. **焦点が正しい**
   - クライアントが本当に知りたいこと
   - 自分にとってのメリット

2. **説得力がある**
   - 同じ情報でも、印象が全く違う
   - 「安心感」を直接的に伝えられる

3. **プロフェッショナル**
   - 価値提供に集中している
   - 対等な関係を保っている

---

## 同じ情報でも伝え方で変わる

完全成果報酬型の説明：

### NG: 提供者視点
```
完全成果報酬型 = 成果が出なければ報酬ゼロ。
この仕組みで既にリスクを100%負っています。

成果が出なければ、私も報酬を受け取りません。
リスクは私も負います。
```

**問題点**:
- 「私も」「私も」の連呼
- 提供者の苦労アピール
- クライアントは「だから何？」と思う

---

### OK: クライアント視点
```
完全成果報酬型 = 成果が出なければ費用は一切発生しません。
御社は初期投資ゼロ、ランニングコストもゼロで始められます。

成果が出なければ、御社の費用負担は一切ありません。
まずは診断を受けていただき、改善可能性を確認してから判断いただけます。
```

**良い点**:
- クライアントのメリットが明確
- 安心感を直接的に伝えている
- 「だから何？」に答えている

---

## ギルトトリップの危険性

### ギルトトリップとは？

相手に罪悪感を感じさせることで、行動を引き出そうとする手法。

**ビジネスでの例**:
- 「こんなに頑張っているのに...」
- 「私もリスクを負っているのだから...」
- 「私も損をするのだから...」

### なぜダメか？

1. **対等な関係ではない**
   - ビジネスは価値の交換
   - 罪悪感で動かすのは不健全

2. **長期的な信頼関係を損なう**
   - 一時的には効果があるかもしれない
   - 後で「騙された」と感じる

3. **プロフェッショナルではない**
   - プロは価値で勝負する
   - 感情操作は不要

---

## B2Bコミュニケーションの本質

### クライアントが知りたいこと

1. **自社にとってのメリットは何か**
   - コスト削減？売上増加？リスク低減？

2. **どのくらいのリスク・投資が必要か**
   - 初期費用は？月額は？最悪の場合はいくら？

3. **なぜその価格なのか**
   - 透明性、納得感

4. **他社と比べてどうか**
   - 業界標準と比較

### クライアントが興味ないこと

1. **提供者の苦労**
   - それはあなたの問題

2. **提供者のリスク**
   - それもあなたの問題

3. **提供者の感情**
   - ビジネスは価値の交換

## 実践：書き換えの原則

| 提供者視点（NG） | クライアント視点（OK） |
|-----------------|---------------------|
| 私がリスクを負う | 御社のリスクはゼロ |
| 私も報酬を受け取らない | 御社の費用負担は一切なし |
| リスクを私も負います | 御社は安心して始められます |
| リスクヘッジしています | 御社に安心していただくため |
| 私の工数がかかる | 御社の手間はかかりません |
| 私も損をする | 御社は損をしません |

**原則**: 同じ情報でも、「誰にとってのメリット・デメリットか」で焦点を変える

---

## 応用範囲

この考え方は提案書に限らない。

### あらゆるビジネスコミュニケーション

- **メール・LINE**
  - 「私が忙しい」ではなく「御社のご都合は？」

- **打ち合わせ**
  - 「私の経験では...」ではなく「御社にとっては...」

- **価格交渉**
  - 「私の工数が...」ではなく「御社が得られる価値は...」

### プロフェッショナルの姿勢

- **自分の苦労は見せない**
  - クライアントは価値を買っている
  - プロセスではなく結果

- **対等な関係を保つ**
  - 感情操作ではなく、価値で勝負
  - Win-Winの関係

- **常にクライアントにとっての価値を考える**
  - これは何のため？
  - クライアントにどんなメリットがある？

## 自己チェック

文章やプレゼンの後、自問する：

1. **誰の視点で書いているか？**
   - 自分 → NG
   - クライアント → OK

2. **クライアントはこれを読んで何を感じるか？**
   - 「だから何？」→ NG
   - 「これは自分にとって良いことだ」→ OK

3. **感情操作していないか？**
   - 罪悪感を利用している → NG
   - 価値を明確に提示している → OK

---

## まとめ

**ビジネスコミュニケーションの本質は、クライアントの視点に立つこと。**

- 提供者の苦労や犠牲は、クライアントの関心事ではない
- 同じ情報でも、焦点を変えるだけで印象が全く変わる
- プロは価値で勝負し、自分の苦労は見せない
- ギルトトリップは短期的には効果があっても、長期的には信頼を損なう

**常に問う**: これは誰のため？クライアントはどう思う？
