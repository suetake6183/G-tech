# 思考と探求 インボックス

**用途**: 個人思考の生データ（音声メモ、アイデア等）を一時保管

---

## このフォルダの役割

- 音声文字起こしの保管
- 散歩中のアイデアメモ
- ふと思いついた考察
- まだ整理されていない思考

---

## ファイル命名規則

```
YYYY-MM-DD-HHMMSS_[トピック].md

例:
20251118-162527_双方向学習システムの設計.md
20251120-093000_クライアント提案の振り返り.md
```

**タイムスタンプを含める理由**: 複数の音声メモを区別するため

---

## YAMLメタデータ

```yaml
---
type: voice_memo  # or idea / reflection
date: YYYY-MM-DD HH:MM:SS
duration: X分X秒  # 音声メモの場合
topics: [トピック1, トピック2]
related_projects: []  # 関連プロジェクトがあれば
processing_status: raw  # raw / reviewed / processed / archived
potential_knowledge: yes/no  # ナレッジ化の可能性
potential_blog: yes/no  # ブログ化の可能性
---
```

---

## 処理フロー

```
音声文字起こし / アイデアメモ
    ↓
03_思考と探求/_inbox/に配置
    ↓
（任意のタイミングで）
    ↓
/process 実行
    ↓
Claude が提案:
1. トピック別フォルダに移動
2. ブログのネタとして活用
3. ナレッジベースに反映
4. そのまま保管
```

---

## 利用可能なカテゴリ

- `ビジネス哲学/`
- `コミュニケーションスタイル/`
- `意思決定パターン/`
- `学習記録/`
- `自己分析/`

---

## よくある使い方

### 1. 音声メモの文字起こし
```
散歩中に音声録音
    ↓
文字起こし
    ↓
このフォルダに配置
    ↓
後日 /process で整理
```

### 2. アイデアの一時メモ
```
ふと思いついたアイデア
    ↓
すぐにメモ（このフォルダ）
    ↓
後日見直して、価値があれば整理
```

### 3. ブログの種
```
思考メモ
    ↓
/blog で執筆
    ↓
ナレッジベースにも反映
```

---

**原則**: まず記録、整理は後で。思考を失わないことが最優先。
