# ナレッジベース インボックス

**用途**: リサーチ結果や学びを一時保管し、ナレッジ化前の整理場所

---

## このフォルダの役割

- Claude/Cursorでのディープリサーチ結果
- Web検索で見つけた情報
- まだカテゴリが不明確な知識

### 配置判断

**明確なカテゴリがわかる** → 直接カテゴリフォルダへ
```
例: 01_ナレッジベース/提案書作成/2025-11-18_成果報酬型の業界標準_リサーチ.md
```

**カテゴリが不明確** → ここ（_inbox/）へ
```
例: 01_ナレッジベース/_inbox/2025-11-18_XXX_リサーチ.md
```

---

## ファイル命名規則

```
YYYY-MM-DD_[トピック]_リサーチ.md

例:
2025-11-18_成果報酬型の業界標準_リサーチ.md
2025-11-20_Notion活用方法_リサーチ.md
```

---

## YAMLメタデータ

```yaml
---
type: research
date: YYYY-MM-DD
topics: [トピック1, トピック2]
source: Claude / Cursor / WebSearch / 文献
related_projects: []  # 関連プロジェクトがあれば
processing_status: raw  # raw / reviewed / knowledged
---
```

---

## 処理フロー

```
リサーチ実施
    ↓
カテゴリ判断
    ↓
┌─────────┬─────────┐
│明確      │不明確    │
└─────────┴─────────┘
    ↓          ↓
カテゴリ    _inbox/
フォルダ       ↓
              /process
              ↓
        適切な場所へ移動
        or
        ナレッジ化
```
