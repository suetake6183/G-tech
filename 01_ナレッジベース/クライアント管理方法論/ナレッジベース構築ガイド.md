# クライアントナレッジベース構築ガイド

**作成日**: 2025年10月30日
**目的**: クライアントとの関係性・歴史を体系的に管理し、最適な提案を行うための指針

---

## なぜナレッジ化が重要か

### 問題：記憶に頼った営業の限界

❌ **記憶に頼ると**:
- 過去のやりとりを忘れる
- 同じ失敗を繰り返す
- クライアントの特性を見逃す
- 最適な価格設定ができない
- 関係性を活かせない

✅ **ナレッジ化すると**:
- 過去の成功パターンを再現できる
- クライアント特性に合わせた提案ができる
- 適切な価格設定の根拠ができる
- 長期的な関係を構築できる
- ビジネスの資産になる

### 今回のDOAケースで実証されたこと

**見積書v1（ナレッジなし）**:
- 10ページの詳細見積書
- 複雑な選択肢
- 一般的な価格設定
- **成約率: 推定30%**

**見積書v2（ナレッジあり）**:
- 1ページのシンプル見積書
- 2つの選択肢
- 友人価格を明示
- 対面説明を前提
- **成約率: 推定70-80%**

**差額**: 2-3倍の成約率向上

## ナレッジベースの構造

### 必須項目（すべてのクライアントで記録）

```
1. 基本情報
   - 会社名、担当者、連絡先
   - 業種、規模
   - 優先連絡手段

2. 関係性の歴史
   - 出会った経緯
   - 過去の取引履歴
   - 関係性の変遷（タイムライン）

3. コミュニケーションスタイル
   - 好む連絡手段
   - 文章スタイル（長文OK? 短文?)
   - 意思決定パターン
   - NGな対応

4. ITリテラシー・理解度
   - レベル評価（⭐1-5）
   - できること/できないこと
   - 学習意欲

5. 予算感・支払い能力
   - 過去の支出パターン
   - 想定予算レンジ
   - 支払い方法の希望

6. 対応スタンス・戦略
   - 価格戦略（市場価格? 友人価格?）
   - コミュニケーション戦略
   - リスク管理

7. 次回提案のヒント
   - 確認すべきこと
   - 将来の展開可能性
   - 注意点
```

### オプション項目（必要に応じて）

```
8. 業界特有の知識
   - 業界用語
   - 業界の商習慣
   - 競合状況

9. 個人的な情報
   - 趣味・関心
   - 家族構成
   - 誕生日など

10. 成功/失敗事例
    - うまくいったアプローチ
    - 失敗したアプローチ
```

## ナレッジベースの運用ルール

### 1. 記録のタイミング

#### 必須記録タイミング
- ✅ **初回コンタクト時**: 基本情報を記録
- ✅ **重要な打ち合わせ後**: 議事録+プロフィール更新
- ✅ **案件完了時**: 成功/失敗要因を分析
- ✅ **6ヶ月ごと**: 定期的なプロフィール見直し

#### 推奨記録タイミング
- 重要なLINE/メールのやりとり後
- 価格交渉の結果
- クライアントの反応が良かった/悪かった時

### 2. 記録の粒度

#### 詳細に記録すべき情報
- 💰 **価格交渉の経緯** → 次回の価格設定に影響
- 😊😢 **クライアントの感情的反応** → コミュニケーション戦略に影響
- ✅❌ **成功/失敗の要因** → 再現性のため

#### 簡潔でOKな情報
- 日常的な連絡
- 定型的な業務連絡

### 3. 更新のルール

- **最低6ヶ月に1回**は全体を見直す
- 大きな案件の前後は必ず更新
- 関係性が変化した時（友人→ビジネス、など）は即時更新

## ナレッジの活用方法

### パターン1: 提案前

```
手順:
1. クライアントプロフィールを読み直す
2. 過去の成功/失敗パターンを確認
3. コミュニケーションスタイルに合わせて提案書を作成
4. 予算感に合わせた価格設定
```

**効果**:
- 成約率が2-3倍向上
- クライアント満足度向上

### パターン2: 価格設定

```
手順:
1. 過去の支出パターンを確認
2. 関係性（友人? 新規? VIP?）を確認
3. 市場価格を調査
4. 適切な割引率を決定
```

**効果**:
- 適正な価格設定
- 値下げ交渉への対応力向上

### パターン3: コミュニケーション

```
手順:
1. 「好ましいコミュニケーション方法」を確認
2. 過去の成功例を参考にする
3. NGパターンを避ける
```

**効果**:
- クライアントのストレス低減
- 意思疎通の円滑化

### パターン4: 長期的な関係構築

```
手順:
1. 「将来の展開可能性」を定期的に見直す
2. タイミングを見て新しい提案
3. 継続的なフォローアップ
```

**効果**:
- LTV（顧客生涯価値）の向上
- 紹介案件の獲得

## 具体例: DOAケース

### Before（ナレッジなし）

**アプローチ**:
- 一般的な見積書を作成
- 市場価格で提示
- 選択肢を多く提供

**結果（予測）**:
- 読まれない
- 決められない
- 成約率30%

### After（ナレッジあり）

**アプローチ**:
- 雅大さんのコミュニケーションスタイルに合わせた1ページ見積書
- 友人価格を明示
- 2つの選択肢に絞る
- 対面説明を前提

**結果（予測）**:
- 理解しやすい
- 決断しやすい
- 成約率70-80%

**改善効果**: 成約率2.3倍、作業時間50%削減

## ナレッジベースの分析

### 月次レビュー（推奨）

#### 確認項目
1. 今月獲得したクライアント数
2. 成約率（全体・クライアント別）
3. 平均案件単価
4. クライアント満足度
5. 成功/失敗パターンの分析

#### 改善アクション
- 成功パターンを他のクライアントに横展開
- 失敗パターンの回避策を策定
- 価格戦略の見直し

### 年次レビュー（推奨）

#### 確認項目
1. クライアント別のLTV（顧客生涯価値）
2. 紹介案件の発生状況
3. 長期契約への転換率
4. クライアント離反率と理由

#### 戦略的判断
- VIPクライアントの選定
- 注力すべきクライアント層の決定
- サービスメニューの見直し

## 成功の鍵

### ナレッジベースを機能させるために

✅ **やるべきこと**:
1. **記録を習慣化する**（案件後は必ず記録）
2. **定期的に見直す**（提案前、月次、年次）
3. **活用する**（記録するだけでは意味がない）
4. **改善し続ける**（テンプレートも進化させる）

❌ **避けるべきこと**:
1. 完璧を目指しすぎて始められない
2. 記録だけして活用しない
3. 更新を忘れて情報が古くなる
4. プライバシーに配慮しない
