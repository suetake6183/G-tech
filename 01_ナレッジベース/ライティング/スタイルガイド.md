# ライティングスタイルガイド（末武修平版）

**最終更新**: 2025年11月14日
**用途**: Claude Codeが末武修平の文体で文章を書く際の参照

---

## 基本原則

### 1. 結論ファースト
結論→理由→詳細 の順序で書く

**例**:
```
❌ 様々な要因を検討した結果、やはり完全成果報酬型が最も適していると考えられます。

✅ 完全成果報酬型（パターンA）を推奨します。
   理由：御社のリスクが完全にゼロだからです。
```

---

### 2. 簡潔・明快
冗長な表現を避ける

**NG表現**:
- 「〜と考えられます」「〜だと思われます」
- 「結論としては、〜」（見出しで既に明確なら不要）
- 「〜という形で」「〜というような」

**OK表現**:
- 「〜です」「〜します」
- 断定的でシンプル

---

### 3. 適度なカジュアルさ
「です・ます」調だが、堅苦しくない

**バランス**:
- ❌ 過度にフォーマル: 「誠に恐縮ではございますが」
- ❌ 過度にカジュアル: 「マジで」「めっちゃ」
- ✅ 適度: 「正直に申し上げて」「率直に言うと」

---

### 4. 正直な表現
誇張しない、言い切らない

**例**:
- ❌ 「絶対に成果が出ます」
- ✅ 「絶対に成果が出るとは言い切れません」

- ❌ 「100%保証します」
- ✅ 「成果が出た場合のみ報酬をいただきます」

---

## 絵文字の使い方

### 見出しレベル: OK
視認性向上のため、セクション見出しでの使用は可

```markdown
## 📋 はじめに
## 🎯 提案の進め方
## 💰 料金体系
```

### 本文・箇条書き: NG
本文中や箇条書きでの絵文字は使わない

```markdown
❌ - ✅ クライアントのリスクゼロ
❌ - ❌ 成果が出なければ工数すべてが無駄

✅ - クライアントのリスクゼロ
✅ - 成果が出なければ工数すべてが無駄
```

**例外**: 見出し直下で「メリット/デメリット」を対比する場合のみ許容

## B2B文脈での表現

### NG: B2C的表現
```
❌ こんな方におすすめ
❌ あなたにとって
❌ お客様
```

### OK: B2B的表現
```
✅ こんな場合におすすめ
✅ 御社にとって
✅ 貴社、御社
```

## クライアント視点の徹底

### 基本ルール
**提供者の苦労ではなく、クライアントのメリットを書く**

### NG: 提供者視点
```
❌ 私がリスクを100%負っています
❌ 成果が出なければ私も報酬を受け取りません
❌ リスクを私も負います
❌ その分、リスクヘッジしています
```

### OK: クライアント視点
```
✅ 御社のリスクは完全にゼロです
✅ 成果が出なければ御社の費用負担は一切ありません
✅ 御社は初期投資ゼロで始められます
✅ 御社に安心していただくため
```

### 書き換えの原則

同じ情報でも、焦点を変える：

| 提供者視点（NG） | クライアント視点（OK） |
|-----------------|---------------------|
| 私がリスクを負う | 御社のリスクはゼロ |
| 私の工数がかかる | 御社の手間はかかりません |
| 私も損をする | 御社は損をしません |

## 「無料」の扱い

### 原則: 1回述べれば十分

```
❌ 無料診断→無料ヒアリング→無料で現状把握→費用は完全無料
   （何度も繰り返すと安っぽい）

✅ 最初に「完全無料」と明記し、以降は「診断」「ヒアリング」のみ
```

---

## 数字・データの扱い

### 具体的に、ただし過度な精度は避ける

```
❌ リスクを100%負っています
   （「100%」は過剰表現）

✅ リスクを完全に負っています
   （「完全に」で十分）

❌ 約〇〇円程度
   （「約」と「程度」の重複）

✅ 約〇〇円 または 〇〇円程度
```

## 構成の原則

### 1. 冗長な繰り返しを避ける

試算例で既に情報を提示した場合、その直後に同じ内容を繰り返さない

```
❌ 【試算例】
   〜具体的な数字〜

   【メリット】
   〜試算例と同じ内容の繰り返し〜

✅ 【試算例】
   〜具体的な数字〜
   （これで十分。繰り返さない）
```

### 2. 見出しで内容が明確なら、プレフィックス不要

```
❌ ## 推奨案
   結論：パターンAを推奨します。

✅ ## 推奨案
   パターンAを推奨します。
```

## FAQ対応のスタイル

### 誠実で率直に答える

```
Q: 本当に成果が出るか不安です

❌ 絶対に成果が出ます。安心してください。
   （誇張、不誠実）

❌ 成果が出なければ私も報酬を受け取りません。リスクは私も負います。
   （提供者視点、ギルトトリップ的）

✅ 正直に申し上げて、絶対に成果が出るとは言い切れません。
   だからこそ、完全成果報酬型をご提案しています。
   成果が出なければ、御社の費用負担は一切ありません。
   （誠実＋クライアント視点）
```

## 文体の一貫性

### 語尾のバリエーション

同じ語尾を連続させない

```
❌ 〜です。〜です。〜です。
✅ 〜です。〜ます。〜できます。
```

### 箇条書きの統一

- 名詞止めで統一 or 文で統一
- 混在させない

```
❌ - リスクゼロ
   - 透明性が高い方式です
   - データに基づいて判断

✅ - リスクゼロ
   - 透明性が高い
   - データに基づく判断
```

## チェック項目

文章を書いた後、以下を確認：

- [ ] 結論が最初に来ているか
- [ ] 冗長な表現がないか
- [ ] 「私のリスク」等の提供者視点表現がないか
- [ ] 「無料」を2回以上使っていないか
- [ ] 本文中に絵文字（✅❌）を使っていないか
- [ ] B2C的表現（「こんな方」等）がないか
- [ ] 誇張表現（「絶対」「100%」）がないか
