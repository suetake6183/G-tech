## 現在の運用状況

### 使用中のシステム

| 項目 | 詳細 |
|------|------|
| **ソフトウェア** | キングソフト（中国製のExcel互換、安価） |
| **デバイス** | 2万円のタブレット（1年半前購入） |
| **運用歴** | **1ヶ月前まで完全手書き** → 現在デジタル化1ヶ月目 |
| **データ管理** | 月ごとにブック（ファイル）を分けている |
| **シート構成** | 日報（日ごとにシート）、小計表、売掛台帳 |

### 重要な発見：「先月まで手書き」

```
雅大: 「先月まで手書きだっけ俺たち」
雅大: 「これはやばい。それまで全部手書きだよ。計算も全部電卓」
末武: 「だからこう物理的に見ないとわけわかんないんだろうなと思って」
雅大: 「ちょっと慣れてきたかなみたいな。今一ヶ月やったんじゃないですか」
```

**意味**:
- デジタル化してまだ1ヶ月
- 「慣れ親しんだシステム」ではない
- 変更への抵抗は低い可能性
- ただしITリテラシーは非常に低い

## 現在の問題点

### 1. 手作業での転記が多い

**日報 → 小計表 → 売掛台帳の流れが全て手入力**

```
雅大: 「この担当の売り上げは、結局これまた別のに打ち込んで、
      承継とか会計を出してるのよ」
末武: 「それをいっぺんにできるようにできるのか」
雅大: 「できる？」
末武: 「できるできる」
```

**具体的な転記作業**:
- 日報の売上を小計表に手入力
- 売掛金額を売掛台帳に手入力
- 入金があったら売掛台帳を手入力で更新
- **転記ミスのリスク大**

### 2. データ共有が困難

```
末武: 「これって今ここで打ち込んだものを他の場所で見るっていうのが
      ちょっと面倒くさい感じになっちゃってます」
末武: 「物理的にここにいるんであれば、このままのやり方でも
      特に問題はない。だけど、それが面倒となれば...」
```

**現在の共有方法**:
- タブレットを物理的に持ち運ぶ
- データをメールで送る
- Dropbox（遅くて使えない）

### 3. インターネット環境が非常に悪い

```
雅大: 「インターネットを見れないぐらい遅いんです」
雅大: 「光こっちの回線のせいなのか、このタブレットのせいなのか
      わかんないんですけど」
末武: 「今、菅野さんとこの事務所をこの間変えたんだけど、
      これの百倍以上ある」
```

**影響**:
- YouTube再生が頻繁に止まる（店内BGM用）
- Dropboxが使えない
- Googleスプレッドシートは現状では使えない

### 4. タブレットの性能が低い

```
末武: 「このパソコンのエンジン、原付みたいな感じ」
末武: 「これ1個落ちたでしょ。ポチって押してから開くまでが
      結構時間かかるもんね。ちょっとストレスかもしれない」
```

**実測**:
- ファイル1つ開くのに数十秒
- 設定画面表示に時間がかかる
- CPUが非常に低性能

## 要件（やりたいこと）

### 1. 自動転記・自動集計

**最優先要件**

```
雅大: 「日報を打っていけば、キャストさんロットの売り上げが分かる。
      売りかけがいくら残ってるのか分かる。指名本数とか場内本数とか、
      そういったものが、日報を打っていくだけで出る」
```

**自動化したい内容**:
- ✅ 日報入力 → キャスト別小計に自動反映
- ✅ 日報の売掛 → 売掛台帳に自動記録
- ✅ 総売上、動員数の自動集計
- ✅ 指名本数、場内本数の自動集計

### 2. キャスト別の顧客履歴

**特殊な要件**

```
雅大: 「担当さんとお客様が言ってくるじゃないですか。
      それを個別というか、こういう状態にはできたりするんですか」
```

**現在の運用**:
- 日報から手入力で顧客履歴を作成
- 「大河さんという方が、何日に何番卓に来て、
  女の子は誰々で、いくら使った」という履歴

**末武の回答**:
```
末武: 「日報でここら辺のデータを打ってさえくれれば、
      要はデータの見方を変えてるだけですよね。
      それは全然可能だと思います」
```

### 3. 入力方法はできるだけ変えたくない

```
末武: 「できればあんまり変えたくはないよね。多分。
      その打ち方とか」
雅大: 「そうだね」
末武: 「できれば同じような打ち方で、出せる数字が勝手に出てくると」
```

**理由**:
- 現在のフォーマットに慣れてきたばかり（1ヶ月）
- 大きな変更は混乱を招く

### 4. 複数の場所からデータを見たい

```
末武: 「タケルさんが打ったやつ全部見れるみたいな感じになるから。
      ここに来る必要性は一つなくなる」
```

**要望**:
- 雅大さんが自宅や別の場所からデータ確認
- リアルタイムでの確認

## 技術的な制約と課題

### 1. 現在のデバイス

| デバイス | スペック | 状態 | 使用用途 |
|---------|---------|------|---------|
| **タブレット** | 2万円、1.5年前購入 | 非常に遅い | 現在のメイン |
| **Surfaceっぽいパソコン** | 菅野さんから借用 | 比較的高性能 | 未使用（雅大さんの私物？） |

**タブレットの問題**:
- CPU性能が極めて低い
- メモリも少ない可能性
- 「原付エンジン」レベル

**Surfaceの可能性**:
```
末武: 「こっちにいいパソコンあるじゃん」
雅大: 「これはなんか、ゆうかさんがやった...菅野さんのパソコン持ってきて」
末武: 「これで（Googleスプレッドシート）やったら多分組まない（固まらない）と思う」
```

### 2. インターネット環境

**現状**:
- 光回線らしいが、契約内容不明
- 実測速度は楽天モバイル程度（非常に遅い）
- Wi-Fiルーターあり（DOA/DOA FREE）
- 物理的な配線が不明（調査必要）

**問題の原因候補**:
1. 回線契約が古い（10年前の契約？）
2. ルーターが古い・性能が低い
3. タブレットのWi-Fi受信性能が低い

**改善案（末武提案）**:
```
末武: 「インターネットの会社さん変えるのと、ルーターを変えるだけで
      速くなる可能性はある」
末武: 「十倍ぐらいは多分やるのがすぐできると思う」
```

**費用感**:
- 回線契約: 月額3,000-5,000円程度
- ルーター: 1-2万円（一回のみ）
- YouTube Premium: 月額1,000円（オプション）

### 3. ソフトウェア選択の判断

#### オプションA: キングソフトのまま

**メリット**:
- ✅ 現在使っているので慣れている
- ✅ インターネット不要
- ✅ タブレットでも動く（遅いけど）

**デメリット**:
- ❌ データ共有が面倒
- ❌ リアルタイム確認不可
- ❌ 自動化の実装がやや複雑

#### オプションB: Googleスプレッドシート

**メリット**:
- ✅ 複数の場所から同時アクセス可能
- ✅ リアルタイムでデータ確認
- ✅ 自動化がしやすい（末武が使い慣れている）
- ✅ 自動バックアップ

**デメリット**:
- ❌ インターネット環境必須
- ❌ 現在の環境では動作が重い

**末武の判断**:
```
末武: 「インターネットの環境を良くする前提だったら、
      まあこれもありかなと思ったけど、
      この環境だとちょっとおすすめできないよ」
末武: 「さっきのキングソフトのやつのまんまの方が
      まだ使いやすいかもしれない」
```

**最終判断**: インターネット環境改善が前提

## 重要な洞察

### 1. 「1ヶ月前まで手書き」の意味

**ポジティブ**:
- 変更への抵抗が低い（まだ慣れていない）
- どんな改善でも「画期的」に感じる
- 期待値が低い（少しの改善でも満足度高い）

**ネガティブ**:
- ITリテラシーが非常に低い
- トラブル時の対応が困難
- 複雑なシステムは使いこなせない可能性

### 2. インターネット環境が最大のボトルネック

**判断**:
- Googleスプレッドシート導入はインターネット改善後
- 現時点ではキングソフトのまま自動化を実装
- 段階的な移行が必要

### 3. 料金に対する抵抗感の低さ

**観察**:
- 「手間賃ぐらい」という曖昧な表現で合意
- 具体的な金額を聞かない
- 信頼関係が非常に強い

**リスク**:
- 後から「思ったより高い」と感じる可能性
- 明確な見積もりの提示が必要

## 提案の方向性

### Phase 1: 最小限の自動化（キングソフトのまま）

**内容**:
- 日報 → 小計表の自動転記
- 日報 → 売掛台帳の自動記録
- 基本的な集計の自動化

**メリット**:
- インターネット環境不要
- 現在のデバイスで動作
- 最小限の変更

**デメリット**:
- データ共有は依然として面倒
- リアルタイム確認不可

### Phase 2: インターネット環境改善

**内容**:
- 回線契約の見直し
- ルーター交換
- 速度テスト

**費用**:
- 初期: 1-2万円（ルーター）
- 月額: 3,000-5,000円

### Phase 3: Googleスプレッドシート移行

**内容**:
- Phase 1の機能をスプレッドシートで再実装
- リアルタイム共有
- より高度な自動化

**前提条件**:
- Phase 2完了
- Surfaceパソコンの活用
