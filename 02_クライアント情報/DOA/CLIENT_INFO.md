## 📁 フォルダ構造ガイド

このクライアントフォルダの情報格納ルールです。

### 00_CORE/（クライアント共通情報）
**実際の構造**:
- `CLIENT_PROFILE.md` - 詳細なクライアント分析（17年来の関係性、意思決定パターン、ITリテラシー等）
- `analysis/` - 分析資料（クライアント全体に関わるもの）
  - `DOA_関係性再評価.md` - 関係性の深層分析
- `source/` - 原データ
  - `[LINE]雅大.txt` - LINE記録（2019-2025年）
  - `10-23 顧客相談_*.txt` - ヒアリング音声書き起こし
- `99_アーカイブ/` - 旧構造、提案作成過程等

**ここに格納するもの**:
- ✅ クライアントプロフィール、戦略資料、深層分析
- ✅ 全プロジェクト共通の原データ（LINE記録、音声書き起こし）
- ✅ クライアント全体の関係性分析

**格納しないもの**:
- ❌ 特定プロジェクトの見積書・提案書 → 各プロジェクトフォルダへ
- ❌ プロジェクト固有のヒアリング・分析資料 → 各プロジェクトフォルダへ
- ❌ プロジェクト固有の預かり資料 → 各プロジェクトフォルダへ

### プロジェクトフォルダ/
**命名規則**: `[プロジェクト名]`（例: `Excel自動化システム`）

**標準構造**:
```
[プロジェクト名]/
├── 00_INPUT/          # クライアントから受領した資料
│   ├── ヒアリング/
│   ├── 預かり資料/
│   └── 参考資料/
├── 01_OUTPUT/         # クライアントへ提出する成果物
│   ├── クライアント向け/
│   │   ├── 見積書/
│   │   ├── 提案書/
│   │   └── 納品物/
│   └── 内部用/       # 社内資料、分析レポート等
├── 02_ARCHIVE/        # 完了・不採用案件のアーカイブ
└── PROJECT.md         # プロジェクト概要（事業主体: 末武修平）
```

### ファイル命名規則
```
YYYYMMDD_[カテゴリ]_[内容].拡張子

例:
20251023_ヒアリング_DOA要件確認.txt
20251030_見積_Excel自動化システム.md
20251101_提案_システム開発提案書.pdf
```

## 💼 提案パターンと反応（蓄積型）

**目的**: やり取りから得た学びを記録し、次回提案の精度を向上させる
**更新方針**: 会議後や重要なやり取りの後に気づきを追記（1-2行でOK）

### 効果的だったアプローチ
- **対面での説明**: お店で直接会って説明 → 信頼関係を活かせる
- **図解・デモ**: 図を描きながら説明 → 理解しやすい
- **スモールスタート**: 「まず2.5万円でこれやってみよう」 → リスクが小さい
- **段階的な提案**: 「まずこれ」が決めやすい
- **「友人価格」を明示**: 特別感がある、信頼関係を維持

### 避けるべきアプローチ
- **長文の見積書を送りつける**: 読まない、「AI相手にしてるみたい」
- **複雑な選択肢**: 3つ以上だと決められない → 2つに絞る
- **いきなり高額提示**: 躊躇する → 段階的に
- **専門用語連発**: 理解できない → 比喩を使う（「原付エンジン」など）
- **ビジネスライクすぎる**: 距離を感じる → 友人関係を大切に
