# 🎯 SecondBrain システム原則（実用的修正版）

## 🔧 核心原則（絶対遵守）

### **1. Single Source of Truth (SSOT) 原則**
**すべての情報は唯一の場所に存在し、他はそれを参照する**

### **2. ディレクトリ配置の厳格化**
```
90_System/Core/              # 定義ファイルのみ
20_Areas/                    # 生活・仕事エリア
10_Projects/                 # 進行中プロジェクト
30_Resources/                # 参考資料
```

### **3. 重複作成の絶対禁止**
- 同一情報の複数箇所配置禁止
- 日記ファイルの重複配置禁止
- システム定義ファイルの重複作成禁止

## ⚠️ AI動作制約（最小限）

### **必須確認事項（5項目のみ）**
1. **ワークスペース確認**: 作業開始時の `pwd` 確認必須
2. **既存ファイル確認**: 新規作成前の重複チェック必須
3. **正規パス使用**: `SecondBrain/[相対パス]` 形式の統一
4. **日付正確性**: 推測による日付設定禁止
5. **Core保護**: Core内への非定義ファイル配置禁止

### **段階的対応システム**
```
Level 1: 警告ログ → 処理継続
Level 2: 確認要求 → ユーザー判断
Level 3: 処理停止 → エラー詳細表示
Level 4: システム停止 → 重大違反のみ
```

## 🚨 緊急停止条件（重大違反のみ）

以下の場合のみシステム停止:
1. **Core内への不正ファイル作成**
2. **SSOT原則の重大違反**（同一情報の3箇所以上配置）
3. **システム破壊的操作**（ディレクトリ構造の変更）

## 📊 監視システム（軽量版）

### **基本監視**
- 重複ファイル検出（日次）

### **高度監視（オプション）**

- 予防保守（効果測定後に導入）
