# 現状分析と課題整理

## 現状の業務フロー

### 会員管理の現状

1. **入会プロセス**
   - **主な入会経路**:
     - Instagramのライブ配信等を通じての申し込み
     - オフラインの講演会・セミナーからの申し込み
   - **申し込み受付方法**:
     - 久仁子さんのInstagramのDMで連絡
     - 公式LINEへの登録（DMからの案内が必要）
   - **顧客情報取得**:
     - Googleフォームからの入力情報をスプレッドシートに反映
     - 基本情報（名前、インスタアカウント、メールアドレス、メッセージなど）が羅列

2. **会員情報管理**
   - **ヒトラボ会員**:
     - スプレッドシートによる管理
     - インスタグラムの鍵アカウントとLINEのオープンチャットへの招待
     - オープンチャットではニックネームでの参加が多く、本名との照合が困難
   - **ライフ講座参加者**:
     - 別のスプレッドシートで管理
     - ヒトラボとライフの両方に所属する会員は別々のリストに記載

3. **コミュニケーションチャネル**
   - **公式LINE**: 個別連絡、お知らせ配信に使用
   - **LINEオープンチャット**: ヒトラボ会員向けの情報共有
   - **Instagramの鍵アカウント**: ヒトラボ会員向けコンテンツ配信
   - **InstagramのDM**: 個別相談や問い合わせ対応
   - **複数チャネルの問題**: チャネルごとに情報が分散し、管理が複雑化

4. **決済管理**
   - **決済方法**:
     - カード決済（Stripe）
     - 銀行振込（様々な支払いサイクルが存在）
   - **決済状況確認**:
     - Stripeで確認（スプレッドシートと連携なし）
     - 決済エラー時の対応に手間がかかる
     - カード名義と会員名が異なるケースで照合に手間がかかる

5. **退会・解約処理**
   - **現状の退会フロー**:
     - Stripeサブスクを手動で切る
     - オープンチャットとInstagram鍵垢は会員自身で退会
     - 管理側でアカウントの削除確認が困難

### コンテンツ管理の現状

1. **講座内容の配信**
   - **配信方法**:
     - LINEで個別に送付（画像の一斉送信ができない）
     - 過去のコンテンツの見返しが困難
   - **過去の対応**:
     - 以前はWordPressでコンテンツを整理（現在は機能していない）
     - ドメインとサーバーの更新忘れで失効

2. **動画コンテンツ管理**
   - **保存・配信方法**:
     - YouTubeの限定公開設定で共有
     - 過去に動画ストレージサービス（Vimeoなど）を使用した経験あり
     - 個別セッション動画も保存しているが、見返す時間がない

3. **Q&A対応**
   - **よくある質問への回答**:
     - 同じ質問に何度も個別に回答する状況
     - 特に健康・身体の仕組みに関する質問が多い
   - **対応履歴**:
     - 過去どのようなアドバイスをしたかの記録がない
     - 毎回内容を聞き直している状態

4. **コンサル・セッション管理**
   - **予約管理**:
     - LINEとスプレッドシートによる非効率な調整作業
     - 久仁子さんの空き時間と会員の希望のすり合わせに苦労
   - **カルテ管理**:
     - 個別コンサルの内容を振り返る仕組みがない
     - 「カルテのような」会員別の記録システムの必要性

## 主要課題の整理

### 1. 管理・運営の課題

1. **情報の散在と非効率性**
   - 顧客情報、コンテンツ、コミュニケーション履歴が複数のプラットフォームに分散
   - 必要な情報を探すのに時間がかかる
   - 過去のコンテンツを再利用する際に手間がかかる

2. **会員情報管理の複雑さ**
   - 会員増加に伴う管理の煩雑化
   - 会員データの不正確さ（入力ミスや情報の不一致）
   - 複数のアカウント（インスタ、LINE等）と会員の照合が困難

3. **決済関連の課題**
   - 決済エラー時の対応の煩雑さ
   - 支払い状況の一元管理ができていない
   - 多様な支払いサイクル（毎月、3ヶ月、6ヶ月など）の管理の複雑さ

4. **継続的管理の難しさ**
   - 管理能力の自己認識の低さ（久仁子さん自身の課題として認識）
   - 過去の例：WordPressサイトの管理を怠りドメインとサーバーが失効

5. **再現性の低さ**
   - 個人の能力や記憶に依存したプロセス
   - 組織拡大やサポーター育成時の引き継ぎの難しさ

### 2. コミュニケーション戦略の課題

1. **多チャネル問題**
   - 複数のコミュニケーションチャネル（Instagram DM、公式LINE、オープンチャット等）
   - チャネルごとの情報の分断
   - どのチャネルを使うべきかユーザーにとっても分かりにくい

2. **情報発信の非効率性**
   - インスタグラムをメイン媒体として活用しているが、特別なプロモーション施策はあまり実施していない
   - ライブ配信による集客が中心
   - プッシュ型の情報発信はあまり好まない傾向

3. **コミュニケーション履歴の不足**
   - 過去のやり取りや対応履歴の蓄積がない
   - 会員との継続的なコミュニケーションの質の維持が困難
   - メッセージの見落としリスク

### 3. 組織体制の課題

1. **役割分担の不明確さ**
   - 指示出しが苦手（久仁子さん自身の認識）
   - チームメンバー（特にざわみさん）に対して明確なゴールや期待を伝えられていない
   - 「お見合い」のようなミスが発生する状況

2. **人材育成の課題**
   - 個別コンサルティングを提供できる人材の育成に時間がかかる可能性
   - 現状のアプローチは「自分自身の課題解決ができていない人はサポーターになれない」という高いハードル
   - サポーターの育成方法の見直しを検討中

3. **業務委託ベースの組織構造**
   - スタッフは業務委託契約が基本
   - 組織としての拡大に伴う連携や役割分担の明確化が必要
   - 久仁子さん個人への依存度が高い

### 4. 事業拡大に伴う課題

1. **キャパシティの限界**
   - 久仁子さん自身の「時間的な限界」の認識
   - 「未来の忙しさを予防する」という考え方
   - 個人の能力に依存する形の限界

2. **スケーラビリティの確保**
   - 会員数増加に対応できる体制構築
   - 「倍になっても大丈夫かどうか」という視点
   - 現状の仕組みでは拡大に対応できない可能性

3. **物販事業拡大の課題**
   - 在庫管理、受注、発送などの業務効率化の必要性
   - 植物発酵エキスの事業承継と生産拡大における実務面の課題
   - 商品そのものの品質管理（製造上の問題）が最大のリスク

4. **講座運営の効率化**
   - ライフ講座の受講希望者増加に対応するための体制構築
   - 講座形式の変更（録画形式への移行、グループディスカッション重視）に伴う実務的な変化
   - コンサル予約の効率化の必要性

### 5. IT関連の課題

1. **IT活用のばらつき**
   - 業務効率化ツールの経験不足
   - ITリテラシーの差異（久仁子さん・たかこさんは試行錯誤で使えるが、会員向けには詳細なマニュアルが必要）
   - 学習スタイル：自分たちでやりながら理解するタイプ

2. **情報セキュリティへの懸念**
   - YouTubeの限定公開での動画共有に懸念
   - より安全な動画共有方法の必要性
   - 顧客情報保護の重要性

3. **システム導入の障壁**
   - 新しいツール・システムへの適応コスト
   - 「便利だからどんどん使っていこう」というアプローチが全ての人に適用できるわけではない
   - ITに馴染みが薄い方々も含めた設計の必要性

## 成功の判断基準

久仁子さんからのヒアリングに基づき、以下を成功の判断基準としています：

1. **時間創出の実現**
   - 特定の仕事に困っているわけではないが、未来の忙しさを予防するための先行投資という位置づけ
   - 「現在の時間の余裕が半年後、1年後も変わっていない状態」を目標
   - 久仁子さんが忙しくなっていなければ投資は成功

2. **具体的な業務改善**
   - コンテンツ管理の改善（受講生がアクセスしやすい形に）
   - 顧客情報の一元管理と活用
   - コンサル履歴の記録・参照システム
   - チームコミュニケーションの効率化

これらの課題と優先順位を踏まえ、Notionを活用した情報管理システムの構築を進めることで、久仁子さんの事業運営の効率化と成長を支援します。
