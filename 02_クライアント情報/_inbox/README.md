# クライアント情報 インボックス

**用途**: クライアントが不明確、または新規クライアント候補のデータを一時保管

---

## このフォルダの役割

### 保管すべきデータ

1. **新規クライアント候補**
   - 問い合わせがあった新規企業
   - まだクライアントフォルダを作るほどではない段階
   - 商談が進んだらクライアントフォルダを作成

2. **クライアント不明のデータ**
   - どのクライアントに関連するか不明確
   - 複数クライアントに関連する可能性がある
   - 後で判明したら適切なフォルダへ移動

3. **プロジェクト不明確なクライアントデータ**
   - クライアントは特定できる
   - しかし既存プロジェクトに関連するか、新規案件か不明
   - 判明後、適切なプロジェクトフォルダへ移動

---

## 配置判断フロー

```
クライアント関連データ
    ↓
クライアント特定できる？
    ├─ YES → プロジェクト特定できる？
    │         ├─ YES → [クライアント]/[プロジェクト]/00_INPUT/
    │         ├─ NO → [クライアント]/_inbox/（オプション）
    │         └─ または [クライアント]/00_CORE/source/
    │
    └─ NO → ここ（02_クライアント情報/_inbox/）
```

---

## ファイル命名規則

```
YYYY-MM-DD_[種別]_[内容]_[企業名].[拡張子]

例:
2025-11-18_問い合わせ_Web制作依頼_ABC株式会社.md
2025-11-20_メール_業界レポート_複数社関連.md
2025-11-22_資料_新規商談_XYZ工務店.pdf
```

**企業名を含める理由**: 後で整理する時に判断しやすくなる

---

## YAMLメタデータ

```yaml
---
type: inquiry / email / document / meeting / other
from: [企業名・担当者名]
date: YYYY-MM-DD
topics: [トピック1, トピック2]
client_status: prospect / existing / unknown  # 見込み/既存/不明
action_required: yes/no
processing_status: raw  # raw / reviewed / processed
potential_project: yes/no  # プロジェクト化の可能性
---
```

---

## 処理フロー

### 新規クライアント候補の場合

```
問い合わせ受領
    ↓
02_クライアント情報/_inbox/ に配置
    ↓
商談進行
    ↓
クライアント化決定
    ↓
1. [クライアント名]/ フォルダ作成
2. CLIENT_INFO.md作成
3. 00_CORE/CLIENT_PROFILE.md作成
4. データを移動
```

### クライアント判明の場合

```
データ受領（クライアント不明）
    ↓
02_クライアント情報/_inbox/ に配置
    ↓
調査・確認
    ↓
クライアント判明
    ↓
/process で適切な場所へ移動提案
    ↓
[クライアント]/[プロジェクト]/00_INPUT/ へ移動
```

---

## `/process` での処理

```
/process
    ↓
Claude: 02_クライアント情報/_inbox/ に3件あります。

       1件目: 2025-11-18_問い合わせ_ABC株式会社.md
       内容: Web制作の問い合わせ

       推奨:
       - 新規クライアントフォルダ「ABC株式会社」を作成
       - プロジェクト「Web制作」を開始

       どうしますか？
```

---

## 個別クライアントフォルダ内の `_inbox/`（オプション）

クライアントは特定できるが、プロジェクトが不明確な場合：

```
02_クライアント情報/
└── 黒澤工務店/
    ├── _inbox/  # オプション（必要に応じて作成）
    ├── 00_CORE/
    └── [プロジェクト]/
```

**代替案**: `00_CORE/source/` を使用

どちらを使うかは、クライアントごとの運用で判断。

---

## 運用ルール

1. **溜めすぎない**: 月1回は `/process` で整理
2. **メタデータ必須**: 後で判断できるように
3. **迅速な判断**: クライアント化が確定したらすぐにフォルダ作成

---

## テンプレート

`/_inbox/_TEMPLATE/` の各種テンプレートを使用：
- EMAIL_TEMPLATE.md
- MEETING_TEMPLATE.md
- 等

---

**原則**: クライアント不明・新規候補はここに。判明したら即座に移動。
