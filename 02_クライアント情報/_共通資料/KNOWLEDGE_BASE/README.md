# KNOWLEDGE_BASE（ナレッジベース）

## 🧠 このフォルダの役割

プロジェクト横断的な**知識・ノウハウ・ベストプラクティス**を蓄積します。

---

## 📁 サブフォルダ

### プロンプト集/
AIプロンプトのサンプル、ベストプラクティス

**内容例:**
- スライド資料作成プロンプト
- 議事録作成プロンプト
- コーチングプロンプト
- 提案書作成プロンプト

**使い方:**
- Claude Codeで作業する時の参考に
- プロンプトをコピーして、プロジェクトに応じてカスタマイズ

### マニュアル/
業務マニュアル、ガイドライン

**内容例:**
- 施工マニュアル
- 業務フローガイド
- 品質管理チェックリスト
- トラブルシューティング

**使い方:**
- 新しいプロジェクトで参照
- 施工時の確認用

### 営業ノウハウ/
営業トーク、提案手法、成功事例

**内容例:**
- 営業トークスクリプト
- 提案手法
- 成功事例・失敗事例
- 営業会議録（過去のナレッジ）

**使い方:**
- 新規クライアントへの提案前に参照
- 営業チームで共有

### 教材/
研修資料、教育コンテンツ

**内容例:**
- AI講座資料
- 社員研修資料
- 技術教育コンテンツ

**使い方:**
- 社内研修で活用
- クライアント向け教育サービスで活用

---

## 💡 使い方のコツ

### 1. ナレッジの蓄積
プロジェクトで得た学びを、ここにドキュメント化

**例:**
```
プロジェクト完了後
↓
「このプロンプトが効果的だった」
↓
KNOWLEDGE_BASE/プロンプト集/[カテゴリ]_[内容].md に保存
```

### 2. 再利用
新しいプロジェクトで同じ課題に直面した時、ここを参照

**例:**
```
「見積書の書き方が分からない」
↓
KNOWLEDGE_BASE/マニュアル/見積書作成ガイド.md を参照
```

### 3. 継続的改善
ナレッジを使ってみて、改善点があれば更新

---

## 📝 ファイル命名規則

### プロンプト集
- `[用途]_[詳細].md`
- 例: `スライド作成_プロンプト.md`

### マニュアル
- `[業務]_マニュアル.md`
- 例: `看板施工_マニュアル.md`

### 営業ノウハウ
- `[カテゴリ]_[内容].md`
- 例: `初回提案_トークスクリプト.md`

### 教材
- `[講座名]_[内容].md`
- 例: `AI講座_プロンプト基礎.md`

---

## ⚠️ 重要なルール

### ✅ ここに入れるもの
- 複数のプロジェクトで再利用可能な知識
- 汎用的なノウハウ
- ベストプラクティス
- テンプレート化できるプロンプト

### ❌ ここに入れないもの
- 特定クライアント固有の情報 → クライアントフォルダへ
- 特定プロジェクト固有の分析 → プロジェクトフォルダへ
- 個人的なメモ → 個人フォルダへ

---

## 🔄 ナレッジの追加フロー

### プロジェクト完了時
1. プロジェクトの `01_OUTPUT/内部用/` で学びを整理
2. 再利用可能な部分を抽出
3. KNOWLEDGE_BASE に追加
4. README.mdを更新

### 定期的な見直し
- 月次: 新しいナレッジの追加
- 四半期: 古いナレッジの更新・削除

## 📊 ナレッジの価値

このフォルダの真の価値は：
- **時間削減**: 同じことを何度も調べない
- **品質向上**: 過去の成功パターンを再利用
- **ノウハウ資産化**: 個人の知識を組織の資産に
- **継続的改善**: 常にベストプラクティスを更新
