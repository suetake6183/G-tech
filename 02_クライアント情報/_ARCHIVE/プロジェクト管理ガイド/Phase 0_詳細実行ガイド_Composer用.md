## 背景と目的

### 現在の課題
- SKコームと末武個人の両方で事業活動を行っている
- クライアントが被ることがある（例: 黒澤工務店）
- どちらの事業主体で取引しているか不明確
- SKコームフォルダに会社基本情報と個別案件が混在（302ファイル）

### 解決策
- **_自社情報/** フォルダを作成し、事業主体の基本情報を集約
- 各クライアントフォルダには **CLIENT_INFO.md** で事業主体を明記
- 各プロジェクトフォルダには **PROJECT.md** で事業主体を明記

### 新しい構造イメージ
```
/02_クライアント情報/
├── _自社情報/                    # 新規作成
│   ├── SKコーム/                 # 会社基本情報
│   │   ├── 会社情報/
│   │   ├── ブランド資産/
│   │   ├── 財務/
│   │   ├── 契約雛形/
│   │   ├── 人事・労務/
│   │   ├── 管理資料/
│   │   └── README.md
│   └── 末武修平/                 # 個人事業基本情報
│       ├── 事業情報/
│       └── README.md
├── SKコーム/                      # 個別案件のみ残る
│   ├── 00_CORE/                  # 軽量化される
│   ├── 顧問契約/
│   └── CLIENT_INFO.md             # 新規作成
├── 黒澤工務店/
│   ├── AI導入・DX推進/            # PROJECT.md（事業主体: 末武修平）
│   ├── 看板工事_20XX/             # PROJECT.md（事業主体: SKコーム）
│   └── CLIENT_INFO.md             # 取引形態を記載
└── ...
```

## ⚠️ エラーハンドリング

### エラー1: git mvでファイルが見つからない

```
fatal: not under version control, source=SKコーム/00_CORE/ロゴ
```

**対処法:**
1. `git status` で未追跡ファイルを確認
2. 未追跡ファイルの場合は `mv` コマンドを使用
```bash
mv "SKコーム/00_CORE/ロゴ" "_自社情報/SKコーム/ブランド資産/"
```

### エラー2: 移動先フォルダが存在しない

```
mv: cannot move to '_自社情報/SKコーム/会社情報/': No such file or directory
```

**対処法:**
1. Step 1のディレクトリ作成コマンドを再実行
2. `mkdir -p` で強制的に作成

### エラー3: ファイルが既に存在

```
mv: overwrite '_自社情報/SKコーム/財務/ゆうちょダイレクト.png'?
```

**対処法:**
1. 既存ファイルを確認（`ls -la`）
2. 古いファイルであれば上書き（`y`）
3. 新しいファイルであれば移動元を確認
