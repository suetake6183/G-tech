# PERSONAL_個人顧客

## 概要
個人顧客からの直接発注案件です。
法人クライアントに比べて情報量は少ないものの、丁寧な対応が求められます。

## クライアント一覧

### 野口悦子様
- **案件**: リフォーム工事
- **特記事項**: 2025年案件

### 渡邉みい
- **案件**: プロジェクト関連（写真データのみ）
- **データ**: 写真4枚

### ムラカミリョウ
- **案件**: プロジェクト関連（写真データのみ）
- **データ**: 写真13枚

## 特徴
- **情報密度が低い**: 写真や基本的な契約書類のみの場合が多い
- **単発案件**: 継続的な取引よりも単発プロジェクトが中心
- **紹介経由が多い**: 既存クライアントや知人からの紹介

## 管理方針
1. **シンプルな構造**: 過度に複雑な構造は不要
2. **基本情報の記録**: 連絡先、案件概要、見積・請求書は必ず保管
3. **写真の整理**: ビフォー・アフター写真は貴重な資産
4. **フォローアップ**: 工事完了後のアフターフォローで信頼関係構築

## データ構造
```
顧客名/
├── 見積・契約/
├── 写真/
│   ├── ビフォー/
│   └── アフター/
└── その他/
```

フラットな構造で十分。情報量に応じて柔軟に対応。
