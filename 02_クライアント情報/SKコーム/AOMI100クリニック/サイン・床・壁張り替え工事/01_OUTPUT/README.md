# 01_OUTPUT（アウトプット）

## 📤 このフォルダの役割

プロジェクトのために**作成した書類**を格納します。

- **受け手別に整理**: 誰に見せる書類かで分類
- **誤送信防止**: フォルダ構造で受け手が明確
- **バージョン管理**: 同じ書類の複数バージョンを時系列で管理

---

## 📁 サブフォルダの使い分け

### クライアント向け/
クライアントに提出・共有する書類

**提案書/**
- プロジェクト提案書
- デザイン提案

**見積書/**
- 見積書
- 料金表

**報告書/**
- 進捗報告
- 完了報告
- 工事概要書

**命名規則**: `YYYYMMDD_[書類種別]_[内容]_v[バージョン].md`
- 例: `20251107_工事概要とスケジュール_v1.md`

---

### 外注先向け/
協力業者・外注先に提出する書類

**依頼書/**
- 制作依頼書
- 発注依頼書

**仕様書/**
- 詳細仕様書
- 技術要件

**命名規則**: `YYYYMMDD_【[業者名]向け】[内容].md`
- 例: `20251106_【制作会社向け】看板デザイン提案依頼書.md`

---

### 内部用/
社内で使う書類（クライアント・外注先には見せない）

**要件定義/**
- 要件定義書
- 仕様検討メモ

**進捗管理/**
- 進捗管理表
- チェックリスト
- 詳細ヒアリング項目（内部用）

**分析レポート/**
- 案件分析
- 収益性分析
- リスク分析

**命名規則**: `YYYYMMDD_[内容]_内部用.md`
- 例: `20251104_打ち合わせヒアリング項目_詳細版（内部用）.md`

---

## 💡 使い方のコツ

### バージョン管理
同じ書類を更新する場合：
- `_v1.md`, `_v2.md` でバージョン番号を付ける
- または日付で区別（推奨）: `20251106_提案書.md`, `20251107_提案書.md`

### PDFの扱い
- マークダウンで作成 → PDF化した場合、両方とも保存
- 例: `提案書.md` と `提案書.pdf` が同じフォルダに共存してOK

### 最終版の明示
- ファイル名に `_最終版` を付ける
- または `_確定版` `_提出版`

---

## ❌ ここに入れないもの

- クライアントから受け取った資料 → `00_INPUT/` へ
- 古いバージョン・不要になった書類 → `02_ARCHIVE/` へ
