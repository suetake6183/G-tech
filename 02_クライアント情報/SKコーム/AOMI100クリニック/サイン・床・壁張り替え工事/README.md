# プロジェクトテンプレート

## 🎯 このテンプレートの目的

新規プロジェクトを開始する時に、この構造をコピーして使います。

- **統一された構造**: すべてのプロジェクトが同じ構造
- **AIが理解しやすい**: Claude Codeがプロジェクト状況を即座に把握
- **迷わない**: どこに何を入れるか明確

---

## 📋 使い方

### 1. 新規プロジェクト開始時
```bash
# このフォルダをコピー
cp -r "_TEMPLATE/PROJECT_TEMPLATE" "クライアント名/プロジェクト名"
```

### 2. PROJECT.mdを編集
- [ ] プロジェクト名を記入
- [ ] 作成日を記入
- [ ] 現在の状況を記入
- [ ] 関係者情報を記入
- [ ] 予算・スケジュールを記入

### 3. プロジェクトを進める
- 情報を収集したら → `00_INPUT/` に格納
- 書類を作成したら → `01_OUTPUT/` に格納（受け手別）
- 不要になったら → `02_ARCHIVE/` へ移動
- **常に PROJECT.md を更新**（進捗、TODO、最新の動き）

---

## 📁 フォルダ構造

```
プロジェクト名/
├── PROJECT.md                     ⭐ 最重要：プロジェクトの司令塔
├── README.md                      このファイル
│
├── 00_INPUT/                      📥 収集した情報
│   ├── 議事録/
│   ├── ヒアリング/
│   ├── 写真・現場資料/
│   ├── ロゴ・デザイン素材/
│   └── 参考資料/
│
├── 01_OUTPUT/                     📤 作成した書類
│   ├── クライアント向け/
│   │   ├── 提案書/
│   │   ├── 見積書/
│   │   └── 報告書/
│   ├── 外注先向け/
│   │   ├── 依頼書/
│   │   └── 仕様書/
│   └── 内部用/
│       ├── 要件定義/
│       ├── 進捗管理/
│       └── 分析レポート/
│
└── 02_ARCHIVE/                    📦 不要になったファイル
```

## 🤖 AIとの連携

### Claude Codeに依頼する時
「[プロジェクト名]のPROJECT.mdを読んで、現在の状況を教えて」
「[プロジェクト名]のクライアント向け提案書を作成して」
「[プロジェクト名]の進捗をPROJECT.mdに更新して」

### AIが読むべきファイルの優先順位
1. **PROJECT.md** - 全体像を把握
2. **00_INPUT/** - 背景情報を収集
3. **01_OUTPUT/** - 既存の書類を確認

## ✅ 日常の運用ルール

### 毎日
- [ ] 何か作業したらPROJECT.mdの「最新の動き」を更新
- [ ] TODOを完了したらチェック

### 週次
- [ ] PROJECT.mdの「現在の状況」を見直し
- [ ] 次週のTODOを整理

### プロジェクト完了時
- [ ] PROJECT.mdに完了日・成果を記入
- [ ] 学び・改善点を記入
- [ ] 不要なファイルを02_ARCHIVE/へ

---

## 🎓 ベストプラクティス

### ファイル命名規則
- **日付を先頭に**: `YYYYMMDD_内容.md`
- **受け手を明示**: `【クライアント向け】` `【業者向け】` `（内部用）`
- **バージョン管理**: `_v1`, `_v2` または日付で区別

### PROJECT.mdの更新頻度
- 理想: 毎日
- 最低: 週1回
- 重要な決定があった時は即座に

### フォルダは増やさない
- 基本構造は変えない
- 特殊な事情があれば検討
