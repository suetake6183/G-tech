# 00_INPUT（インプット）

## 📥 このフォルダの役割

プロジェクトを進めるために**収集した情報**を格納します。

- **読み取り専用的な性質**: 基本的に「集めたもの」を保管
- **参照用**: 書類を作成する時の元ネタ
- **AIが読む**: Claude Codeがこのフォルダを読んでプロジェクトの背景を理解します

---

## 📁 サブフォルダの使い分け

### 議事録/
- 打ち合わせの議事録
- 会議音声のトランスクリプト
- 決定事項の記録

**命名規則**: `YYYYMMDD_打ち合わせ_[内容].md`

### ヒアリング/
- ヒアリングシート
- 要件のメモ
- クライアントの希望・要望

**命名規則**: `YYYYMMDD_ヒアリング_[内容].md`

### 写真・現場資料/
- 現場写真
- 図面
- 既存物の資料

**命名規則**: `YYYYMMDD_[場所]_[内容].jpg`

### ロゴ・デザイン素材/
- クライアントから提供されたロゴ
- デザイン素材
- ブランドガイドライン

**命名規則**: そのまま（クライアントから提供された名前）

### 参考資料/
- その他の参考情報
- 競合調査
- 技術資料

---

## 💡 使い方のコツ

1. **時系列で整理**: 日付をファイル名の先頭に付ける
2. **元ファイルは保持**: クライアントから受け取ったものは原本のまま保存
3. **加工版は01_OUTPUTへ**: 編集・加工したものはアウトプット側へ

---

## ❌ ここに入れないもの

- 自分で作成した書類 → `01_OUTPUT/` へ
- 古くなった資料 → `02_ARCHIVE/` へ
