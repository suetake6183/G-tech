## 🚨 根本課題：PDCAサイクルの不在

### 現状（トランスクリプトより抽出）

#### 問題1: 決めたことが実行されない
```
「タスクを忘れる問題が常態化している」
「いつまでに・誰が・何を、が不明確」
「次回会議で振り返られない」
```

**実態**:
- 会議で決定 → 実行確率は推定30%程度
- タスクの70%が「忘れられている」可能性

#### 問題2: 実行されても検証されない
```
「ルールを変えても効果検証がない」
「課題解決のためにルール改変したが、
 実際に課題は解決したのか確認されない」
```

**実態**:
- ルール変更 = 目的ではなく、手段
- **目的は「課題解決」なのに、それが測定されていない**
- 効果がなくてもルールだけ残る

#### 問題3: フィードバックが上がってこない
```
「指示されたことはやるが、フィードバックしない」
「ルール変えたけど意味ないですという報告が上がってこない」
「主体性がない」
```

**実態**:
- 従業員は言われたことはやる（真面目）
- しかし「やってみてどうだったか」を報告しない
- 経営層が「詰める」必要性が常態化

### これが引き起こす問題

1. **経営層の負担増**
   - 細部まで管理せざるを得ない
   - 「詰める」作業に時間を取られる
   - 戦略的な仕事に時間を使えない

2. **亜希さんの負担増**
   - リマインド業務が常態化
   - 「あれどうなった？」の確認業務
   - 本来業務に時間を使えない

3. **組織の成長停滞**
   - 試行錯誤から学べない
   - 改善サイクルが回らない
   - 同じ問題が繰り返される

4. **100年企業への道が遠のく**
   - 属人化が解消されない
   - 組織的な学習ができない
   - 自律的な成長ができない

## 🎯 本提案の目的

**「PDCAサイクルが回る組織」への変革**

### 具体的には

1. **Plan（計画）**: 会議で決めたことが明確（いつまでに・誰が・何を）
2. **Do（実行）**: タスクが忘れられず、確実に実行される
3. **Check（検証）**: 実行した結果、効果があったかを測定
4. **Act（改善）**: 効果がなければ改善、あれば継続・展開

### これが実現すると

- ✅ 経営層の「詰める」作業が80%削減
- ✅ 亜希さんのリマインド業務がゼロに
- ✅ 試行錯誤から学び、改善が加速
- ✅ 組織が自律的に成長
- ✅ 100年企業への確かな一歩

### 柱1: タスクが忘れられない仕組み

#### 現状の問題
- 会議で決めたことが口頭だけ
- 誰が何をいつまでにするか不明確
- 次回会議で振り返られない

#### 解決策: シンプルPDCAダッシュボード（Notion）

**仕組み**:
1. 会議の最後10分で必ず記録
   - いつまでに（期限）
   - 誰が（担当者）
   - 何を（タスク内容）
   - どの会議で決定したか

2. 自動リマインド（Zapier または Google Calendar）
   - 期限3日前に担当者へ通知（LINE/メール）
   - 期限当日に未完了なら黒澤社長・亜希さんへ通知
   - 次回会議の24時間前に前回決定事項を一覧化

3. 会議の最初10分で必ず振り返り
   - 前回決定事項が表示される
   - 「できた/できなかった」を確認
   - できなかった理由を簡単に記録

**期待効果**:
- タスク完了率：30% → 80%（目標）
- 亜希さんのリマインド業務：月10時間 → 0時間
- 「あれどうなった？」の確認業務がゼロに

**投資額**: 10万円（構築・設定・3ヶ月伴走）
**期間**: 2週間で構築 → 3ヶ月で定着

**ツール比較検討**:
- 詳細は [ツール比較表](03_ナレッジベース/ツール比較表_タスク管理・PDCA支援.md) 参照
- 結論：Notion継続使用（既存活用のメリット大）
- ただし自動リマインドは外部ツールで補完

### 柱2: 効果検証が自動的に行われる仕組み

#### 現状の問題
- ルールを変えても効果測定されない
- 「課題は解決したか？」が確認されない
- 効果がなくてもルールだけ残る

#### 解決策: PDCA記録テンプレート

**仕組み**:
1. ルール変更時に必ず記録（所要時間：3分）
   - 【P】何を変えるか（ルール変更内容）
   - 【P】なぜ変えるか（解決したい課題）
   - 【P】どうなれば成功か（効果測定の基準）
   - 【P】いつ効果を測るか（測定日）

2. 測定日に自動リマインド
   - 「今日は○○のルール変更の効果測定日です」と通知
   - 会議で必ず議題に上げる

3. 効果測定を会議で実施（10分）
   - 【C】実際どうだったか（効果測定の結果）
   - 【A】次のアクション（継続/修正/中止）
   - 記録を残す

**具体例**:

```
【事例】看板設置日の入力ルール

【P】ルール変更: 看板管理表に「設置日」を入力する
【P】課題: 看板の設置確認に手間がかかる
【P】成功基準: 設置確認の問い合わせが月10件→0件
【P】測定日: 2週間後の会議

【C】結果: 問い合わせが月10件→2件に減少（80%削減）
【A】次のアクション: 継続。ただし入力漏れ対策も検討
```

**期待効果**:
- 「なぜこのルールがあるのか分からない」がゼロに
- 効果のないルールが淘汰される
- 改善のサイクルが回り始める

**投資額**: 8万円（テンプレート作成・運用設計・伴走）
**期間**: 1週間で構築 → 3ヶ月で定着

---

### 柱3: フィードバックが自然に上がってくる文化

#### 現状の問題
- 指示されたことはやるが、報告しない
- 「やってみてどうだったか」が上がってこない
- 主体性を期待しているが、ない

#### 解決策: 主体性を「期待」せず「仕組み」で補完

**重要な認識転換**:
> 「主体性がないから報告しない」のではなく、
> 「報告する仕組みがないから報告しない」

**仕組み**:
1. **会議フォーマットの変更**
   - 従来：「報告事項」「議題」「決定事項」
   - 新：「前回の振り返り」「報告事項」「議題」「決定事項」「次回の確認事項」

2. **前回の振り返り**（会議の最初10分、必須）
   - 前回決定したタスク：「できた/できなかった」
   - できた → Why: なぜできた？（成功要因）
   - できなかった → Why: なぜできなかった？（阻害要因）
   - 実施したルール変更：「効果あった/なかった」
   - 効果あった → 継続・展開
   - 効果なかった → Why: なぜ効果がなかった？→ 改善策

3. **聞かれたことに答えるだけでOK**
   - 主体的に報告する必要はない
   - 経営層が聞く → 社員が答える
   - これだけでPDCAが回り始める

4. **徐々に主体性が芽生える**
   - 慣れてくると、聞かれる前に報告するようになる
   - 「これ、効果なさそうです」という声が上がり始める
   - 3-6ヶ月後、主体性のある社員が現れる

**期待効果**:
- 「詰める」必要性が大幅削減
- 問題の早期発見
- 組織的な学習が始まる
- 主体性のある社員が育つ

**投資額**: 7万円（会議フォーマット設計・ファシリテーション研修・伴走）
**期間**: 1週間で設計 → 3ヶ月で定着

- 必要最小限の機能のみ

- 工数削減が目標に達しない場合は改善策を即座に実施
- 最悪でも「学び」は残る（次の打ち手の材料）
