## 📖 重要キーワード（初心者向け解説）

### 1. プロジェクトナレッジ

**簡単に言うと**: AIに会社独自のルールを教える「カンニングペーパー」

**具体例**:

```
【AIに教える情報】
・人名: 高橋一美（たかはし かずみ）← 「高橋一」と略さない
・役職: 代表取締役
・業界用語: 「配筋（はいきん）」= 鉄筋を配置すること
・よくある間違い: 「亜希」→ 正しくは「あき」
```

**効果**: 最初は誤字だらけでも、使うたびに賢くなる（人間の新入社員と同じ）

---

### 2. 半自動化

**簡単に言うと**: AIに下書きを任せて、人間が最終チェックする分業体制

**比喩**:
- ❌ 完全自動化 = 新人にすべて任せっきり（危険）
- ✅ 半自動化 = 新人が8割やって、ベテランが2割確認（安全で効率的）

**具体例（議事録作成）**:

```
【作業時間の比較】
従来: 2時間（全て手作業）
半自動化: 30分（AI処理20分 + 人間の確認10分）
→ 75%の時間削減！
```

### 3. 二段階認証（2FA / MFA）

**簡単に言うと**: 玄関の鍵が2つあるようなもの

**🔒 セキュリティレベル比較**

| 方式 | 安全性 | 説明 | 推奨度 |
| --- | --- | --- | --- |
| **パスワードのみ** | ⭐☆☆☆☆ | 鍵1個だけ（危険） | ❌ 非推奨 |
| **SMS認証** | ⭐⭐⭐☆☆ | パスワード + 携帯SMS | ⚠️ 2025年は非推奨 |
| **認証アプリ（TOTP）** | ⭐⭐⭐⭐☆ | パスワード + Google Authenticator等 | ✅ 推奨 |
| **セキュリティキー** | ⭐⭐⭐⭐⭐ | パスワード + 物理キー | ✅ 最強 |

**⚠️ 重要な更新情報（2025年）**:
従来のSMS認証は**SIMスワップ詐欺**（携帯番号を乗っ取る犯罪）のリスクがあるため、**認証アプリ（TOTP）への移行が推奨**されています。

**🔧 推奨設定**:
1. **メイン**: Google Authenticator / Microsoft Authenticator（認証アプリ）
2. **バックアップ**: 予備コードを印刷して金庫保管
3. **避けるべき**: SMSのみに依存

**具体的な設定方法**:

```
【手順】
1. 各サービスの「セキュリティ設定」を開く
2. 「二段階認証」または「2FA」を選択
3. 「認証アプリ」を選択（SMSではなく）
4. Google Authenticatorアプリでバーコードをスキャン
5. 表示された6桁のコードを入力して完了
```

## セクション1: AI活用による議事録作成の実践と課題

会議の音声データをAIで自動的に文字起こしし、議事録として構造化する取り組みが実施されている。使用ツールはChatGPTやClaudeで、音声ファイルをアップロードすると、話者の識別、発言内容の要約、タスクの抽出までを自動で行う。

この運用で最大の課題は固有名詞の誤認識である。人名が「高橋一美」を「高橋一」と略されたり、「亜希」が誤って表記されるなど、毎回15-20分程度の修正作業が発生している。これを改善するため、プロジェクトナレッジという仕組みを導入している。これは人名の正式表記、読み仮名、よく間違えられる表記、役職、責任範囲などを体系的に記録したドキュメントで、AIに事前に読み込ませることで認識精度を向上させる。

ツールの選択として、業界用語の理解や日本人に馴染む表現という観点では、マナスというサービスが評価されている。ジェンスパークは海外的なデザインになりがちとの指摘がある。

タスク管理の面では、議事録に「誰がいつまでに何をするか」が記載されても、次回会議で「忘れていました」という事態が頻発している。議事録を作成すること自体は実現できているが、それを定期的に振り返り、実行状況を追跡する運用が定着していないことが課題として浮かび上がっている。

音声入力による修正指示の効率化も議論されており、タイピングが苦手な場合は音声で「高橋一ではなく高橋一美です」と修正指示を出し、漢字部分だけ手直しする方法が提案されている。

## セクション2: ファイル管理システムとアクセス権限の設計

社内のファイル共有システムとして、現在はNASを使用している。共有フォルダと個人フォルダが混在しており、基本的には全員が他人のファイルも閲覧可能な状態である。機密性の高い経営資料などは、バックアップサーバーの特定フォルダに保存するか、デスクトップに置くという対応が取られているが、本質的な解決にはなっていない。

この問題について、社内IT担当者に相談したところ「フォルダごとのアクセス制限はできない」という回答があった。しかし、講師はこの回答に疑問を呈している。企業向けのファイル共有システムでアクセス権限管理ができないことは、現代のセキュリティ要求水準からすると考えにくい。できないとすれば、現在のシステムやサービスの限界であり、他のソリューションを検討すべきだという主張である。

代替案として提案されているのがGoogle Workspaceである。月額課金制でランニングコストは発生するが、フォルダ単位での詳細なアクセス権限設定が可能で、導入も比較的容易である。一方、NASは初期費用のみで済むメリットがあるが、容量不足により3-5年周期で大容量機器への買い替えが必要になり、長期的なコスト計算が重要である。

セキュリティの観点からは、物理的な機器であるNASは盗難リスクがある一方、Googleのデータセンターは厳重に管理されているため、むしろクラウドの方が安全という見方も示されている。

IT担当者の選定基準についても議論され、「できない」で終わらせるのではなく、複数の選択肢を提示し、それぞれのコストと機能を比較検討できる提案ができる人材が望ましいとされている。

## セクション3: ネットワークインフラの問題と改善策

契約上は1Gbpsの光回線だが、実測値はプロバイダー経由で300Mbps、社内無線LANでは100Mbps程度しか出ていない状況である。NTTの担当者は「プロバイダーまでは保証できるが、その先は保証外」という説明をしており、これに対して強い不満が表明されている。

### 🔍 【初心者向け】ネットワーク速度低下の原因と診断方法

**📊 速度低下の典型パターン**:

```
契約: 1Gbps（1000Mbps）
  ↓
プロバイダー経由: 300Mbps（70%減）
  ↓
社内Wi-Fi: 100Mbps（90%減）← 異常！
```

**⚠️ 重要**: 契約値の10分の1は明らかに異常。「仕方ない」で済ませてはいけません。

---

### 🛠️ 自分でできる速度診断チェックリスト

### ✅ チェック1: LANケーブルの規格確認

```
【確認方法】
LANケーブルに印字されている文字を見る

❌ Cat5 または CAT5 → 最大100Mbps（古い）
⚠️ Cat5e → 最大1Gbps（ギリギリOK）
✅ Cat6 または Cat6A → 最大10Gbps（推奨）
✅ Cat7 → 最大10Gbps（最良）
```

**💡 対策**: Cat5を見つけたら即交換（1本500円〜）

---

### ✅ チェック2: Wi-Fiルーターの規格確認

```
【Wi-Fi規格と速度】
❌ Wi-Fi 4 (802.11n) → 最大600Mbps（2009年）
⚠️ Wi-Fi 5 (802.11ac) → 最大6.9Gbps（2014年）
✅ Wi-Fi 6 (802.11ax) → 最大9.6Gbps（2019年〜）
✅ Wi-Fi 6E → 最大9.6Gbps + 混雑に強い（2021年〜）
```

**💡 対策**: Wi-Fi 4なら買い替え必須（1-3万円）

---

### ✅ チェック3: 接続方式の確認

```
【プロバイダー接続方式】
❌ PPPoE方式 → 混雑しやすい（古い）
✅ IPoE方式（IPv6） → 混雑しにくい（新しい）
```

**確認方法**:
1. プロバイダーの契約書を見る
2. または「確認くん」サイトでIPv6対応を確認
→ https://www.ugtop.com/spill.shtml

**💡 対策**: PPPoEならIPoE対応プロバイダーに変更検討

---

### ✅ チェック4: MTU値の調整（VPN使用時）

```
【MTU（Maximum Transmission Unit）とは】
= 1回で送れるデータの最大サイズ

VPN使用時はMTU値の不一致でデータが分割され、
速度が大幅に低下することがある
```

**確認方法（Windows）**:

```
1. コマンドプロンプトを開く
2. 「netsh interface ipv4 show subinterfaces」と入力
3. MTU列を確認（通常1500）
```

**💡 対策**: VPN使用時は1400-1450に調整すると改善する場合あり
（IT担当者に相談推奨）

---

| 対策内容 | 期待される改善率 | 費用 | 難易度 |
| --- | --- | --- | --- |
| LANケーブル交換 | 50-100% | 数千円 | ⭐☆☆ 簡単 |
| Wi-Fiルーター更新 | 30-80% | 1-3万円 | ⭐⭐☆ やや簡単 |
| IPoE方式への変更 | 30-50% | 月額+200円程度 | ⭐⭐⭐ 普通 |
| MTU値調整 | 20-50% | 0円 | ⭐⭐⭐ 普通 |
| プロバイダー変更 | 50-100% | 初期費用あり | ⭐⭐⭐ 普通 |

### 🎯 具体的な改善ステップ

**ステップ1（自分でできる）**:
1. LANケーブルをCat6に交換
2. ルーターを再起動
3. 速度測定サイト（Fast.com）で効果確認

**ステップ2（IT業者と相談）**:
1. IPoE方式への変更を依頼
2. Wi-Fiルーターの更新を検討
3. MTU値の調整を依頼

**ステップ3（それでも改善しない場合）**:
1. プロバイダーの変更を検討
2. セキュリティ機器の設定見直し
3. 専門業者のセカンドオピニオンを取得

---

### 💬 IT業者への質問例

「できない」で終わらせない質問の仕方:

❌ 悪い質問:
「速度が遅いんですが、なんとかなりませんか?」
→ 「仕様です」で終わる

✅ 良い質問:
「現在100Mbpsしか出ていません。以下の点を確認・提案してください:
1. LANケーブルの規格は適切か
2. IPoE方式に変更可能か
3. 代替プロバイダーの比較見積もり
4. それぞれの改善率の見込み」
→ 具体的な回答を引き出せる

---

この問題の原因として、セキュリティソフトウェアによる通信のチェック処理が挙げられている。確かにセキュリティ対策により速度低下は発生するが、**大手企業では同様のセキュリティを保ちながらも実用的な速度を実現している**ため、現状の10分の1という速度は異常であるという認識が示されている。

**🔐 セキュリティと速度の両立は可能**:
- 適切な設定で20-50%の速度改善が期待できる
- WireGuardプロトコル等の新技術活用で高速化
- 近距離サーバーの選択で改善

ネットワーク速度が遅いと、社内ネットワーク上のファイルを開く、ファイルを保存する、クラウドサービスにアクセスするといった日常業務すべてに影響が出る。特にパワーポイントのような大容量ファイルを社内サーバーから読み込む際に顕著な遅延が発生している。

改善策として、プロバイダーの変更、セキュリティソフトウェアの見直し、社内ネットワーク機器の更新などが考えられる。ただし、これらの判断には専門知識が必要であり、社内に技術的な判断ができる窓口となる人材が不在である点が課題となっている。

外部のIT業者に依存する場合、その業者が本当に最適な提案をしているか検証できる能力が社内に必要である。年間管理料を支払っているにもかかわらず「できない」という回答だけで終わる状況に対し、**セカンドオピニオンを求める必要性**が議論されている。

## セクション4: パスワード管理とセキュリティ対策

パスワードの保存場所が散在している問題が指摘されている。iPhone、パソコン、Google Chromeなど、それぞれが「パスワードを保存しますか?」と尋ねてくるため、ユーザーの自覚なくバラバラの場所に保存されている。その結果、「普段は入れるのに今日は入れない」「パスワードが見つからない」といった混乱が生じている。

### 🚨 【危険】やってはいけないパスワード管理

**❌ 絶対NG（危険度MAX）**:

```
1. メモ帳・Excelに平文で保存
   → ファイルが流出したら全アカウント乗っ取られる

2. パスワードの使い回し
   → 1つ漏れたら全サービスが危険

3. 付箋に書いてモニターに貼る
   → 誰でも見られる（オフィス泥棒のカモ）

4. 単純なパスワード
   ❌ 「password123」「会社名2025」
   → 数秒で破られる
```

**⚠️ リスク大（やや危険）**:

```
1. ブラウザの自動保存のみに依存
   → PCが盗まれたら終わり

2. スマホのメモアプリに保存
   → クラウド同期で流出リスク
   → 実際の被害事例あり（ネットバンキング不正送金）
```

### ✅ 【推奨】2025年のパスワード管理ベストプラクティス

### 📌 基本方針（3つの柱）

```
1. 強固なパスワード（12文字以上、英数字記号混在）
```

## 理解度確認Q&A

### Q1: AI活用による議事録作成で最も時間がかかる作業は何か。また、その改善策として「プロジェクトナレッジ」とは何か説明せよ。

**A1:** 最も時間がかかるのは固有名詞、特に人名の誤認識を修正する作業である。AIは音声から「高橋一美」を「高橋一」と認識するなど、毎回15-20分程度の修正が必要になっている。プロジェクトナレッジとは、このような誤認識を防ぐため、人名の正式表記・読み仮名・よく間違えられる表記・役職・責任範囲・業界用語などを体系的に文書化したものである。これをAIに事前に読み込ませることで、組織固有の用語や人名の認識精度が向上し、修正作業が削減される。使用するたびにフィードバックを加えて更新することで、継続的に品質が改善される仕組みである。

---

### Q2: 現在のファイル管理システム（NAS）の問題点を3つ挙げ、Google Workspaceへの移行によってどのように解決されるか説明せよ。

**A2:** 問題点は以下の3つである。第一に、フォルダごとのアクセス権限設定ができないため、機密情報も含めて全員が閲覧可能な状態になっている。第二に、容量不足により3-5年周期で大容量機器への買い替えが必要で、その都度データ移行作業が発生する。第三に、物理的な機器のため盗難や破損のリスクがある。Google Workspaceへの移行により、フォルダ単位での詳細なアクセス権限設定が可能になり、職位や部署に応じた情報管理が実現できる。容量も十分に提供され（月15TB程度）、クラウドサービスのためデータセンターで厳重に管理される。ただし月額課金制でランニングコストが発生するため、長期的なコスト比較が必要である。

---

### Q3: 契約上1Gbpsの回線なのに実測100Mbpsしか出ない原因として考えられることを説明し、この状況に対する講師の見解を述べよ。

**A3:** 原因として、プロバイダー側の帯域制限とセキュリティソフトウェアによる通信チェック処理が挙げられる。NTTからプロバイダーまでは1Gbpsで届いているが、プロバイダーから社内までで300Mbpsに低下し、さらにセキュリティ機器やWi-Fiを経由して100Mbpsまで低下している。講師の見解は、セキュリティによる速度低下は一定程度やむを得ないが、契約値の10分の1という状況は異常であるというものである。大手企業でも同様のセキュリティを保ちながら実用的な速度を実現しているため、プロバイダーの変更やセキュリティソフトの見直しを検討すべきである。「そういうもの」という説明だけで終わらせず、具体的な改善策を複数提示できる業者を選ぶべきだと主張している。

---

### Q4: パスワード管理の現状の問題点を説明し、二段階認証とワンパスワードがどのようにセキュリティを向上させるか述べよ。

**A4:** 現状の問題は、パスワードがiPhone、パソコン、Google Chromeなど複数の場所に無自覚に保存され、どこに何があるか把握できていないことである。さらに、メモ帳に平文で保存している場合、そのメモが流出すれば全アカウントが危険にさらされる。二段階認証は、ログインIDとパスワードに加えて**認証アプリ（TOTP方式）**で生成される6桁のコードを要求する方式で、パスワードが漏洩しても物理的に端末を持っていなければアクセスできないため、セキュリティが大幅に向上する。**なお、従来のSMS認証は2025年非推奨となっている**（SIMスワップ詐欺のリスクのため）。ワンパスワードは、マスターパスワード1つで複数の複雑なパスワードを管理でき、権限者ごとに閲覧可能な範囲を設定できるサービスである。社員と経営者で見られる情報を区別できるため、組織的なセキュリティ管理が可能になる。

---

### Q5: 講義全体を通じて、AI活用による「完全自動化」ではなく「半自動化」が推奨されている理由を、具体例を挙げて説明せよ。

**A5:** 理由は、現在のAI技術では組織固有の文脈や正確性を100%担保できないため、人間による確認と修正が不可欠だからである。議事録作成では、AIが音声を文字起こしし、タスクを抽出するところまでは自動化できるが、固有名詞の誤認識や文脈の取り違えが発生する。そのため、人間が15-20分程度で修正作業を行う「半自動化」の形式を取っている。完全自動化を目指すと、誤った情報がそのまま確定してしまうリスクがある。一方、半自動化であれば、AIに大部分の作業を任せつつ、人間が最終チェックすることで、効率と品質の両立が可能になる。この考え方は、業務プロセス全般に適用でき、「AIは作業の大部分を担当し、人間は判断と確認に集中する」という役割分担が、現実的かつ効果的なAI活用の形だと示されている。
