## 📚 講義全体の要約

> 本講義では、AI（ChatGPT、Plaud）を活用して「誰が・いつまでに・何をする」を明確化する議事録と、そこから逆算するタスク管理の設計を学びました。ポイントは「いきなり完成させず、まずAIに提案させて段階的に改善する」こと。プロジェクト機能、カスタムテンプレート、オートフローを組み合わせ、録音→文字起こし→要約→配信までを自動化する実務ワークフローを構築します。
> 

<aside>
💡

狙い：会議の“記録”ではなく“前進”を生む。次回会議の振り返り・進捗確認・ルールの効果検証まで一気通貫で回す。

</aside>

## 🔑 重要キーワード（必要なものだけ開く）

### 🧠 プロンプトエンジニアリング

- まず提案させる→評価→修正→汎用化のサイクル
- 要件は「担当・期限・具体的タスク」を最優先で明示

### ✅ アクションアイテム

- 誰が、いつまでに、何をするのかが一目で分かる粒度
- 「担当者不明」「期限不明」も明示して曖昧さを残さない

### 📁 プロジェクト機能（ChatGPT）

- カスタムプロンプト＋参考資料（社員名簿等）で精度向上
- 名前の表記揺れや誤認識を低減

### 🤖 AIエージェント（Manas）

- 複数ツールを自動選択し作業代行（資料作成・分析など）

### 🧩 カスタムテンプレート（Plaud）

- 業界用語や社内ルールを組み込んだ定型化

### 🔁 オートフロー

- 録音条件→テンプレ自動選択→メール送信までを自動実行

---

## 🧪 実践ステップ｜プロンプト改善ミニワーク

1. 目的を一文で定義する
    - 例：次回会議に向けて、担当と期限が確定したアクションアイテムを一覧化する
2. 必須要件を列挙する
    - 担当者、期限、タスク、背景メモ、未確定ラベル（担当者不明／期限不明）
3. AIにまず提案させる
    - 出力形式見本を渡す（下のテンプレ参照）
4. 評価して修正指示を出す
    - 抜け漏れ、表現、汎用性をチェック
5. 汎用化する
    - 固有名詞や一回限りの指示を排除し、再利用できる形に

<aside>
🧩

出力テンプレ（コピペ用）

</aside>

```
# アクションアイテム
- [ ] タスク：
  - 担当：
  - 期限：
  - 背景・補足：
  - 未確定：担当者不明／期限不明 の場合は明示
```

## 🤖 自動化の要点（Plaud / オートフロー）

- カスタムテンプレートに業界用語と社内用語を登録
- 録音時間で条件分岐（例：0分〜5時間で全件対象）
- 無制限プラン以外は承認制で不要要約を抑制
- Gmail送信を自動実行し配信ミスを防止

<aside>
🛠️

精度を上げる工夫

</aside>

- 担当不明は「不明」、日付なしは「期限不明」と記載
- 欠席者情報も記録して把握漏れを防ぐ

## ❓ 理解度確認Q&A（よくある疑問）

### Q1 なぜまず提案させるのか？

- 汎用化しやすく、組織固有の要件に合わせて短時間で微調整できるため

### Q2 社員名簿を入れる意義は？

- 名前の誤認識が減り、修正工数を圧縮できるため

### Q3 汎用性を持たせる理由は？

- どの会議でも再利用でき、運用負荷を下げられるため

### Q4 オートフロー設定の注意点は？

- 録音条件の網羅と、プラン制限時の承認フロー設計

### Q5 主体性に頼らずタスク管理を回すには？

- PMの役割を明確化し、期限設定と進捗確認をルール化する

## 🧩 講義での追加確認ポイント

- 参加者と欠席者を明記する運用
    - 冒頭で参加者と欠席者を口頭で読み上げると誤認識が減る
    - 出力では「参加者」「欠席者」を分けて記載
- 担当者・期限が不明な場合の表記
    - 担当者不明／期限不明 を明示し、曖昧さを残さない
- 表形式のタスクリストを標準化
    - 「担当」「期限」「タスク」「背景・補足」を最低列として固定

### 💬 出力を安定させるプロンプト雛形（コピペ用）

```
以下の会議文字起こしから、次回会議でそのまま使える要約とタスクリストを作成してください。出力要件は厳守してください。

# 出力要件
1) 会議概要（日時、テーマ、参加者、欠席者）
2) 前回からの進捗確認（できた／未了と理由）
3) 今回決定事項（箇条書きで簡潔に）
4) アクションアイテム（表形式）
	- 列：担当｜期限｜タスク｜背景・補足
	- 担当・期限が不明な場合は「担当者不明」「期限不明」と明記
	- 推測で特定しない
5) 未解決の課題・注意点

# 追加ルール
- 固有名詞は聞き違いに注意し、確証がない場合は保留（不明表記）
- 要約文は一文を短く、主語と動詞を明確に
```

### 📋 「表形式で」の指示例

- 「アクションアイテムは表形式で出力してください。列は『担当｜期限｜タスク｜背景・補足』に固定してください。」

## 🔧 ChatGPT（プロジェクト）設定の具体手順

1. プロジェクトを作成し、上記のプロンプト雛形を「指示」に登録
2. 参考資料を追加
    - 社員名簿（氏名・役職の一覧）
    - よく使う用語・略語（表記ゆれの統一）
3. 改善フロー
    - 初回出力をレビュー → 気になる点を箇条書きで指摘
    - 「汎用性を持たせて再設計して」と依頼し、指示を上書き

---

## 🛠 Plaud（クラウド）カスタムの具体手順

1. ボキャブラリーに業界・社内用語を登録
2. カスタムテンプレート
    - テンプレートガイド → 任意のテンプレート → … → カスタマイズ
    - 先のプロンプト雛形を貼り付け、名称を保存（例：議事録用-標準）
3. 要約作成時
    - 「最要約」実行時にカスタムテンプレートを選び出力

### 🔁 オートフロー（自動配信）

- 条件：録音時間の範囲を広めに設定（例：0分〜5時間）
- アクション：
    - カスタムテンプレートで作成 → 指定のGmailへ自動送信
- ねらい：録音終了→要約→配信までの手動操作をゼロに近づける

---

## 🧭 運用Tips

- 会議冒頭で「参加者・欠席者」を読み上げる
- 日付は「YYYY-MM-DD」で明言（例：前回＝2025-09-16、今回＝2025-09-19）
- 会議末尾で「担当・期限の最終確認」を読み上げて確定させる
- [ ]  会議の目的を1行で定義した
- [ ]  出力テンプレを自社用に微修正した
- [ ]  社員名簿・用語集をプロジェクトに登録した
- [ ]  「担当者不明／期限不明」を明示する運用を決めた
- [ ]  オートフローの条件と承認プロセスを設定した
- [ ]  次回会議で検証する指標（例：未完了率、リードタイム）を決めた
