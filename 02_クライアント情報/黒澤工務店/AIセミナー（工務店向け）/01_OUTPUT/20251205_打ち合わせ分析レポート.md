## 3. 参加者の共通課題・関心

### AI活用レベルの実態

- **録音は既にやっている人が多い**（Plaud.ai等）
- 議事録作成・調べ物・アイデア出しが主な用途
- 「使っているが、正解かわからない」という不安を抱えている

### 共通の課題・関心（5項目）

1. **焦り**: AIに乗り遅れる不安
2. **正解がわからない不安**: 流行りで使っているが、これでいいのか？
3. **社員への展開方法**: 経営者だけでなく社員にも使わせたい
4. **情報セキュリティ**: 経営情報と社員情報の切り分け
5. **言った言わない問題の解決**: 録音→テキスト化→AI要約の流れ

### 具体的に興味を示された活用事例

1. 議事録作成の効率化（30分→5分）
2. 言った言わない問題の解決
3. 営業フィードバック（客観的な分析）
4. 図面の整合性チェック（複数図面の矛盾検出）
5. メール→チャット通知の自動化

## 4. 発言の矛盾・不整合の分析

### 4-1. 松田さん「社員向け」vs「自分はやる気ない」

**発言①**（00:30頃）
> 「いや、違う、違います。違うちゅうか、あの、どっちかしと社員、社員向けでして欲しいなって思ってたんですけど、ま、実務的には使うのはもう社員なんで」

**発言②**（00:58頃）
> 宮森さん「あれ、松田社長はもう自分やる気ないですか？」
> 松田さん「いや、やる気ないっちゅうか。いや、いや、あの、基本、基本、あの、石川会長と一緒ですてやっときやっていうだけですて」

**矛盾点**: 社員にAIを使わせたいと言いながら、自分は「やっとくか」程度のモチベーション。経営者自身が価値を実感していない状態で、社員に導入を促すのは難しい。黒澤さんも「経営者が1番できなきゃダメだよ」と言っていた点と矛盾する。

### 4-3. 宮森さん「セキュリティ懸念」vs「既にPlaud.aiで議事録共有」

**分析**: 周囲からは「先進的」と見られているが、本人は「まだ活かせていない」「AIないとダメの領域に行きたい」と言っている。**相対的には進んでいるが、本人の目標とのギャップがある**。矛盾というより、自己認識と周囲の評価のズレ。

**発言①** 宮森さん（00:28頃）

**発言②** 松田さん（00:11頃）

**分析**: 使っているが「正しく使えているかわからない」という状態。これは**知識の問題ではなく、フィードバックループがない問題**。自分の使い方が合っているか確認する機会がないまま使い続けている。

## 5. 矛盾から見える本質的課題

### 表面と実態の対照表

| 表面的な発言 | 実態 |
|------------|------|
| 「社員に使わせたい」 | 自分が価値を実感できていない |
| 「焦りがある」 | 具体的な行動に移せていない |
| 「セキュリティが心配」 | 何を守りたいか整理できていない |
| 「使っている」 | 「これでいいのか」と不安なまま |

### 矛盾の根本原因

**「AIを使わなきゃいけない」という外部からのプレッシャーと、「自分にとって何が嬉しいのか」という内発的動機がつながっていない。**

参加者は「AIを使っている」と言えるレベルには達しているが、以下の状態にある：

1. **目的と手段の逆転**: 「AIを使うこと」自体が目的化している
2. **成功体験の不足**: 劇的に楽になった・成果が出たという実感がない
3. **フィードバック不在**: 自分の使い方が正しいか確認する機会がない
4. **言語化できていない**: 自分が何に困っていて、何を解決したいのか明確でない

## 6. セミナー設計への示唆

### 従来の想定との差異

**当初の想定**: AIをほとんど触ったことがない人向けの入門
**実態**: 触っているが「これでいいのか」という不安を持つ人が多数

### セミナーで扱うべきテーマの優先度

| 優先度 | テーマ | 理由 |
|-------|-------|------|
| 高 | 今やっていることの「答え合わせ」 | 不安解消のニーズが高い |
| 高 | 「自分にとってのAI活用の意味」の言語化 | 内発的動機の欠如が根本課題 |
| 中 | 議事録効率化の精度向上 | 既にやっているがうまくいかない |
| 中 | 経営情報と社員情報の切り分け方 | 宮森さんの具体的な懸念 |
| 低 | ツールの基本操作 | 既に触っている人が多い |

### 検討すべきセミナー構成案

1. **冒頭**: 参加者の現状確認（何を使っていて、何に困っているか）
2. **前半**: 「それで合ってます」という答え合わせ＋改善ポイント
3. **中盤**: 自分にとってのAI活用の意味を言語化するワーク
4. **後半**: 具体的な精度向上テクニック（議事録、図面チェック等）
5. **締め**: 明日からの具体的アクション
