## 📁 フォルダ構造ガイド

このクライアントフォルダの情報格納ルールです。

### 00_CORE/（クライアント共通情報）
**標準構造**:
```
00_CORE/
├── CLIENT_PROFILE.md        # 詳細なクライアント分析
├── source/                  # 原データ（不変）
│   └── LINE履歴、メール、録音など
├── analysis/                # 分析レポート
│   ├── ナレッジベース/
│   └── DEEP_ANALYSIS_*.md など
└── 99_アーカイブ/           # 旧構造、過去の提案等
```

**ここに格納するもの**:
- ✅ `CLIENT_PROFILE.md`: 経営課題、キーパーソン、深層ニーズの戦略分析
- ✅ `source/`: クライアントとのコミュニケーション原データ（LINE、メール、録音・録画）
- ✅ `analysis/`: クライアント全体の分析レポート、ナレッジベース、提案テンプレート

**格納しないもの**:
- ❌ 特定プロジェクトの見積書・請求書・契約書 → 各プロジェクトフォルダへ
- ❌ プロジェクト固有の資料・会議記録 → 各プロジェクトフォルダへ
- ❌ 基本契約書・NDA → 各プロジェクトの`01_OUTPUT/クライアント向け/契約書/`へ

### プロジェクトフォルダ/
**命名規則**: `[プロジェクト名]`（例: `AI導入・DX推進`）
**注意**:
- プロジェクトフォルダはクライアントフォルダ直下に配置
- 年度別フォルダは使用しない（リポジトリ再構築計画に準拠）
- パスは相対パスで記載（可搬性を考慮）

**標準構造**:
```
[プロジェクト名]/
├── 00_INPUT/          # クライアントから受領した資料
│   ├── 図面/
│   ├── 写真/
│   └── 預かり資料/
├── 01_OUTPUT/         # クライアントへ提出する成果物
│   ├── クライアント向け/
│   │   ├── 見積書/
│   │   ├── 請求書/
│   │   ├── 契約書/
│   │   └── 納品物/
│   └── 内部用/       # 社内資料、分析レポート等
├── 02_ARCHIVE/        # 完了・不採用案件のアーカイブ
└── PROJECT.md         # プロジェクト概要（事業主体を明記）
```

### ファイル命名規則

#### プロジェクト全体の命名規則
```
YYYYMMDD_[カテゴリ]_[内容].拡張子

例:
20250104_見積_案件名.pdf
20250115_契約_基本契約書.pdf
20250201_写真_現場_場所.jpg
20250310_請求_案件名_前受金.pdf
```

#### 00_INPUT/の命名規則（フラット運用）
基本はフラット構造。ファイル名でカテゴリを識別：

```
YYYY-MM-DD_[種別]_[内容].[拡張子]

推奨プレフィックス:
- 会議記録: YYYY-MM-DD_会議_[議題].md
- ヒアリング: YYYY-MM-DD_ヒアリング_[対象者].md
- クライアント提供: クライアント提供_[内容].[拡張子]
- 参考資料: 参考資料_[内容].[拡張子]
- 現場写真: YYYY-MM-DD_写真_[場所].[拡張子]

例:
2025-10-26_会議_AI活用戦略セッション.md
2025-09-15_ヒアリング_社長.md
クライアント提供_ロゴデータ.ai
参考資料_競合分析レポート.pdf
2025-10-10_写真_現場視察.jpg
```

**フォルダ作成のガイドライン**:
- 基本はフラット（サブフォルダなし）で運用
- 同じ種別のファイルが**10件を超えたら**フォルダ作成を検討
  - 例: 会議記録が10件超 → `会議記録/`フォルダ作成
  - 例: 写真が50枚超 → `写真/`フォルダ作成
- フォルダ作成は必須ではなく、管理性向上のための柔軟な判断
