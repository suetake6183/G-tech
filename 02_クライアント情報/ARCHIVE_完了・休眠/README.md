# ARCHIVE_完了・休眠

## 概要
完了した案件、または一定期間連絡がない休眠状態のクライアントです。
完全に削除するのではなく、再活性化の可能性を考慮してアーカイブ保管します。

## クライアント一覧

### ラクリマ
- **ステータス**: 休眠
- **最終コンタクト**: 2024年7月打ち合わせ
- **データ**: 打ち合わせ記録のみ
- **備考**: 7月以降音沙汰なし

## 判断基準
以下のいずれかに該当する場合、このカテゴリに移動：

1. **完了案件**: プロジェクトが完了し、今後の予定がない
2. **休眠状態**: 6ヶ月以上連絡がない
3. **情報が少ない**: 打ち合わせのみで成約に至らなかった案件

## 管理方針
1. **定期的な見直し**: 四半期ごとにアーカイブを見直し
2. **再活性化の可能性**: 年末年始、年度末等に再コンタクトを検討
3. **データの保管**: 削除せず、必要最小限の情報を保管
4. **学びの抽出**: なぜ休眠したのか、次回に活かせる学びを記録

## 再活性化のチェックポイント
- [ ] 最終コンタクトから1年経過したら再度連絡
- [ ] 休眠理由を分析（価格、タイミング、ニーズ不一致等）
- [ ] 再提案の準備（新サービス、価格改定等）

## データ構造
```
クライアント名/
├── README.md              # 休眠理由、経緯、再活性化のヒント
├── 過去のやりとり/
└── 参考資料/
```

再活性化時にすぐに状況を思い出せるよう、README.mdに経緯を記録することを推奨。
