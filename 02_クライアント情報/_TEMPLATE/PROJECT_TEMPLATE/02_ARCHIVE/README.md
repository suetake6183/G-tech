# 02_ARCHIVE（アーカイブ）

## 📦 このフォルダの役割

プロジェクト中に**不要になったファイル**を格納します。

- **削除はしない**: 将来参照する可能性があるため
- **検索対象外**: AIや検索時にノイズにならないよう分離
- **履歴として保持**: 意思決定の経緯を振り返る時に使う

---

## 📁 整理方法

### 日付別フォルダで整理（推奨）
```
02_ARCHIVE/
├── 20251106_初期提案（不採用）/
│   ├── 提案書_v1.md
│   └── 見積書_v1.pdf
├── 20251110_旧デザイン案/
│   └── デザイン案A.jpg
└── 20251120_中間ファイル/
    └── 作業中メモ.md
```

### カテゴリ別フォルダも可
```
02_ARCHIVE/
├── 不採用提案/
├── 旧バージョン/
└── 中間作業ファイル/
```

---

## 🗂️ アーカイブするタイミング

### こんな時にアーカイブ
- ✅ 提案が不採用になった
- ✅ 新しいバージョンができて古い版が不要になった
- ✅ 方針変更で使わなくなった資料
- ✅ プロジェクト完了後、日常的に見る必要がなくなったファイル

### アーカイブしないもの
- ❌ 最終版・確定版の書類（01_OUTPUTに残す）
- ❌ 収集した元情報（00_INPUTに残す）
- ❌ PROJECT.md（常にルートに残す）

---

## 💡 使い方のコツ

### アーカイブ時のメモを残す
フォルダ名や中に `README.md` を作って、なぜアーカイブしたか記録：

```markdown
# アーカイブ理由
- 不採用理由: 予算オーバー
- 代替案: 20251107_提案書_v2.md を採用
```

### プロジェクト完了時
プロジェクトが完全に終わったら、プロジェクトフォルダ全体を：
- クライアントフォルダ直下の `_ARCHIVE/` へ移動
- またはプロジェクトフォルダ内に `_完了.md` マーカーファイルを作成

---

## 🔍 検索のコツ

アーカイブ内を検索したい時：
- Finderで `02_ARCHIVE/` 内を指定して検索
- Claude Codeに「アーカイブ内も含めて検索して」と明示
